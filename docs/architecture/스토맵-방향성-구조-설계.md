# 스토맵(Stomap) 방향성 구조 설계

> "지도 위에 스토리를 남기자"  
> 1인 개발, 위치 기반 기록/발견 앱  
> 작성일: 2026-06-17  
> 상태: 제품 방향 재정의 / BM은 후순위

---

## 1. 핵심 정체성

스토맵은 네이버지도처럼 장소를 저장하는 앱도 아니고, 당근처럼 동네 피드를 보는 앱도 아니다.

스토맵의 중심은 **장소에 남긴 기록이 지도 위에서 살아나는 경험**이다.

### 제품 한 줄 정의

> 스토맵은 내가 다녀온 장소와 지금 주변의 이야기를 지도 위에 기록하고 발견하는 앱이다.

### 스토맵이 가져야 할 감정

- 지도탭: "내 주변에 어떤 이야기가 있지?"
- 스토리 남기기: "이 장소에 지금의 기록을 남겨야지."
- 내 지도: "내가 남긴 기록들이 나만의 지도가 되었네."

### 경쟁 앱과의 차이

| 구분 | 네이버지도 | 당근 | 스토맵 |
|---|---|---|---|
| 중심 가치 | 장소 정보 / 저장 / 길찾기 | 동네 소식 / 거래 / 커뮤니티 | 장소 기반 기록 / 발견 / 회고 |
| 저장 대상 | 가고 싶은 매장, 장소 | 게시글, 동네 활동 | 내가 남긴 장소의 이야기 |
| 지도 위 표현 | 저장 마커, 장소 POI | 지역 기반 피드 보조 | 기록 자체가 지도 위 주인공 |
| 핵심 행동 | 검색, 저장, 길찾기 | 글 보기, 거래, 소통 | 기록 남기기, 주변 이야기 발견, 내 지도 보기 |

---

## 2. 큰 구조 원칙

### 화면 역할을 섞지 않는다

초기에는 지도탭과 내 지도탭을 완전히 분리한다.

- 지도탭에 `내 글 보기`를 넣지 않는다.
- 내 글과 내 기록은 My탭의 `내 지도`에서 본다.
- 나중에 실제 사용 흐름을 보고 지도탭에 내 글 보기 기능을 합칠지 판단한다.

### 지도는 같아도 의미는 달라야 한다

둘 다 지도 화면을 쓰더라도 목적이 다르다.

- 지도탭: 남의 스토리와 주변의 순간을 발견하는 곳
- 내 지도: 내가 남긴 기록을 소유하고 회고하는 곳

### BM보다 제품 구조가 먼저다

카테고리 제한, 광고, 구독 같은 BM은 지금의 우선순위가 아니다.

먼저 사용자가 "내 기록이 지도에 쌓인다"는 가치를 느껴야 한다. 이 가치가 약한 상태에서 결제벽을 세우면 스토맵의 핵심 경험을 스스로 깎게 된다.

## 3. 콘텐츠 타입 구조

스토맵의 기록은 크게 세 타입으로 나눈다.

### 사진 스토리

사진이 있거나 오래 남길 의도가 있는 기록이다.

- 지도탭에서 발견 대상이 된다.
- 내 지도에 쌓이는 핵심 기록이다.
- 이미지 마커를 그대로 사용한다.
- 카테고리가 있다면 My탭 내 지도에서 마커 테두리에 카테고리 색상을 입힌다.

### 숏츠 스토리

짧은 영상으로 남기는 장소 기록이다.

빠른 기간 안에 추가할 핵심 콘텐츠 타입으로 보고, 처음부터 제품 구조에 포함한다.

- 사진 스토리처럼 오래 남는 기록이다.
- 지도탭에서 발견 대상이 된다.
- 내 지도에 쌓이는 핵심 기록이다.
- 지도 위에서는 영상 썸네일 기반 마커를 사용한다.
- 사진 마커와 구분되도록 작은 재생 아이콘 또는 영상 배지를 붙인다.
- 카테고리가 있다면 My탭 내 지도에서 마커 테두리에 카테고리 색상을 입힌다.

숏츠 스토리는 별도 SNS 피드가 아니라 **장소에 남긴 짧은 영상 기록**이어야 한다.

### 순간 기록

사진 없이 텍스트만 남긴 48시간 기록이다.

- 지도탭에서 현재 주변의 생동감을 만드는 요소다.
- 공개 지도에서는 48시간 후 보이지 않는다.
- My탭의 지도 마커에는 기본 노출하지 않는다.
- My탭 안에서 `내가 남긴 순간` 같은 별도 리스트/버튼으로 확인할 수 있게 한다.

### 타입별 역할

| 타입 | 지도탭 | 내 지도 | 보관 방식 |
|---|---|---|---|
| 사진 스토리 | 주변 이야기로 발견 | 내 지도 마커로 표시 | 내 기록으로 지속 보관 |
| 숏츠 스토리 | 주변 영상 기록으로 발견 | 내 지도 마커로 표시 | 내 기록으로 지속 보관 |
| 순간 기록 | 48시간 동안 지도 위 말풍선/텍스트 마커로 표시 | 지도에는 기본 미표시 | 별도 리스트에서 확인 |

## 4. 지도탭 설계

지도탭은 **발견하는 화면**이다.

사용자가 앱에 들어오자마자 "지도 위에 사람들이 남긴 이야기가 있다"는 느낌을 받아야 한다.

### 핵심 역할

- 내 주변 스토리를 발견한다.
- 순간 기록을 통해 현재 주변의 생동감을 느낀다.
- 사진 스토리, 숏츠 스토리, 텍스트 순간 기록을 지도 위에서 탐색한다.

### 1차 개선 방향

현재 선택 카드, 확장 카드, 좌우 스와이프 구조는 유지한다.

추가할 것은 지도 화면의 첫 진입 발견 레이어다.

#### 발견 레이어

하단에 `내 근처 이야기` 카드 덱을 둔다.

- 앱 진입 시 바로 보인다.
- 좌우 슬라이드로 주변 스토리를 훑을 수 있다.
- 이때 지도 마커는 카드 슬라이드에 따라 움직이지 않는다.
- 사용자가 카드를 직접 선택하거나 마커를 선택했을 때 기존 선택 카드 흐름으로 들어간다.

#### 추천 필터

지도탭은 발견 중심이므로 필터도 내 기록이 아니라 주변 발견 기준으로 둔다.

- 주변
- 인기
- 최신
- 순간 기록

`내 글 보기`는 1차에서 넣지 않는다.

### 유지할 기존 구조

#### 마커 선택 카드

현재 구조를 유지한다.

- 마커 선택 시 하단 카드가 올라온다.
- 좌우 스와이프로 옆 스토리를 볼 수 있다.
- 스와이프 시 연결된 옆 마커/스토리 흐름을 유지한다.

#### 확장 카드

현재 구조를 유지한다.

- 상세 내용과 댓글을 보기 위해 확장한다.
- 지도 위 바텀시트 문법을 유지한다.
- 전체 화면으로 완전히 분리되는 느낌보다 "지도 위 기록을 자세히 본다"는 느낌을 유지한다.

### 지도탭에서 피해야 할 것

- 내 글 보기 기능을 메인 필터로 두지 않는다.
- My탭의 카테고리 색상 테두리를 지도탭에서 강하게 쓰지 않는다.
- 지도탭을 게시글 관리 화면처럼 만들지 않는다.

## 5. 스토리 남기기 탭 설계

가운데 탭은 `공유하기`가 아니라 **스토리 남기기**로 바꾼다.

`공유하기`는 SNS에 퍼뜨리는 느낌이 강하고, 스토맵의 핵심 행동인 "장소에 기록을 남긴다"는 의미가 약하다.

### 핵심 역할

- 현재 위치나 선택한 위치에 기록을 남긴다.
- 사진이 있으면 스토리로 남긴다.
- 짧은 영상이 있으면 숏츠 스토리로 남긴다.
- 사진이 없으면 48시간 동안 보이는 순간 기록으로 남긴다.

### 문구 방향

#### 기본 타이틀

`공유하기` 대신 다음 방향을 사용한다.

- 스토리 남기기
- 이 장소에 기록 남기기

#### 사진 있는 경우

- "사진과 함께 이 장소의 기록이 지도에 남아요."
- "이 장소의 이야기를 스토리로 남겨보세요."

#### 영상 있는 경우

- "짧은 영상으로 이 장소의 분위기를 남겨보세요."
- "이 장소의 순간을 숏츠 스토리로 남겨보세요."

#### 사진 없는 경우

- "사진 없이 남긴 기록은 지도에 48시간 동안 보여요."
- "지금 이 장소의 순간을 짧게 남겨보세요."

### 현재 구조 유지 범위

작성 시트의 큰 구조는 유지한다.

- 제목은 선택
- 내용은 필수
- 사진 추가
- 영상 추가
- 위치 확인
- 공개 지도 노출 안내

1차 작업에서는 구조를 크게 바꾸기보다 명칭과 문구를 스토맵답게 정리한다.

## 6. My탭 / 내 지도 설계

My탭은 기존 마이페이지가 아니라 **내 지도**가 되어야 한다.

내 지도는 "내가 남긴 기록들이 쌓여 나만의 지도가 되었다"는 소유감과 회고감을 주는 화면이다.

### 핵심 역할

- 내가 남긴 스토리를 지도 위에서 본다.
- 내가 남긴 숏츠 스토리를 지도 위에서 본다.
- 카테고리별로 내 기록을 필터링한다.
- 카테고리 색상을 통해 기록의 결을 한눈에 본다.
- 내가 남긴 순간 기록은 별도 리스트로 확인한다.

### 화면 첫인상

지도 자체가 주인공이어야 한다.

- 기존 프로필 카드 중심 화면에서 벗어난다.
- 지도 위에 내 이미지/영상 마커가 보인다.
- 상단에는 간결한 프로필/통계/설정만 둔다.
- 하단에는 카테고리/최근 기록/내가 남긴 순간 진입 영역을 둔다.

### 내 지도에서 보여줄 요소

#### 상단 오버레이

- 닉네임 또는 "OO의 지도"
- 전체 스토리 수
- 방문/기록 지역 수
- 설정 버튼

#### 지도 영역

- 내 사진 스토리의 이미지 마커를 표시한다.
- 내 숏츠 스토리의 영상 썸네일 마커를 표시한다.
- 숏츠 스토리는 사진 마커와 구분되도록 작은 재생 아이콘 또는 영상 배지를 붙인다.
- 카테고리가 있는 스토리는 마커 테두리에 카테고리 색상을 적용한다.
- 카테고리 색상은 My탭에서 강하게 사용한다.

#### 카테고리 필터

- 전체
- 카페
- 산책
- 여행
- 맛집
- 사용자가 만든 카테고리

필터를 선택하면 해당 카테고리의 기록만 지도에 보인다.

#### 하단 영역

- 내 최근 기록
- 카테고리별 기록 수
- 내가 남긴 순간
- 좋아요한 스토리
- 공지사항 / 설정

기존 마이페이지의 메뉴 기능은 완전히 없애지 않고, 내 지도 하단 또는 설정 영역으로 흡수한다.

### 순간 기록 처리

순간 기록은 내 지도 마커로 기본 표시하지 않는다.

대신 다음 중 하나로 접근하게 한다.

- `내가 남긴 순간` 버튼
- `최근 순간 기록` 리스트 영역
- 설정/기록 관리 안의 별도 리스트

목적은 내 지도를 영구 기록 중심으로 깨끗하게 유지하는 것이다.

## 7. 카테고리 설계

카테고리는 BM 잠금 장치가 아니라 **내 기록 지도의 색상 레이어**다.

### 방향 변경

이전 BM 문서에서 고려했던 `무료 카테고리 N개 제한`은 폐기한다.

카테고리 제한은 기록 앱의 핵심 욕구와 충돌한다.

- 사용자는 기록을 더 잘 정리하고 싶어서 카테고리를 만든다.
- 카테고리를 막으면 기록을 더 남기고 싶은 흐름도 막힌다.
- 스토맵은 저장 앱이 아니라 기록 앱이므로, 분류 도구는 핵심 무료 기능으로 둔다.

### 확정 방향

- 카테고리 개수 제한 없음
- 카테고리마다 색상 선택 가능
- 이미지 마커는 그대로 유지
- My탭 내 지도에서 이미지 마커 테두리에 카테고리 색상 적용
- 지도탭에서는 카테고리 색상을 강하게 쓰지 않음

### 카테고리 색상 적용 기준

| 화면 | 색상 적용 강도 | 이유 |
|---|---|---|
| 지도탭 | 미적용 또는 약하게 | 발견성이 우선이고 남의 카테고리 색은 중요하지 않음 |
| 내 지도 | 강하게 적용 | 내 기록을 테마별로 보는 핵심 경험 |
| 카테고리 관리 | 강하게 적용 | 색상 선택/수정이 직접적인 관리 대상 |

### 카테고리 관리 화면

관리 화면에서는 다음 기능을 제공한다.

- 카테고리 생성
- 이름 변경
- 색상 변경
- 순서 변경
- 삭제 또는 병합

단, 1차 구현에서는 생성/이름/색상 정도로 시작해도 충분하다.

## 8. 하단 탭 구조

초기 구조는 3탭을 유지한다.

```
[지도]    [스토리 남기기]    [내 지도]
 발견        기록 작성          내 기록
```

### 지도

남의 스토리와 주변 순간 기록을 발견하는 곳이다.

### 스토리 남기기

장소에 기록을 남기는 액션이다.

탭이라기보다 중앙 액션 버튼에 가깝다. 현재 `공유하기`의 역할을 이어받되 이름과 문구를 바꾼다.

### 내 지도

내가 남긴 스토리가 쌓인 개인 지도다.

기존 `My`보다 `내 지도`가 스토맵 정체성에 더 맞다.

## 9. BM 방향

현재는 BM 설계보다 제품 구조를 우선한다.

### 지금 하지 않을 것

- 카테고리 개수 제한
- 광고 노출
- 광고 제거를 중심으로 한 프리미엄
- 기록/공유를 막는 paywall

### 애드몹 방향

애드몹은 준비 상태로만 둔다.

- 지도탭에 광고를 넣지 않는다.
- 스토리 남기기 흐름에 광고를 넣지 않는다.
- My탭 내 지도 경험에 광고를 넣지 않는다.
- 필요하면 나중에 선택형 보상 광고 정도만 실험한다.

### 나중에 검토할 수 있는 BM

카테고리 개수가 아니라 **기록을 더 예쁘게 소유하고 공유하게 해주는 기능**에 과금한다.

후보:

- 고급 지도 테마
- 고해상도 내 지도 이미지 저장
- 연말/월간 기록 리캡
- 프리미엄 마커 스타일
- 숏츠 지도 하이라이트/자동 리캡
- 지도 백업/내보내기
- 공개 프로필/크리에이터 통계

핵심 원칙:

> 기록을 막아서 돈을 받지 않는다.  
> 기록을 더 예쁘게 보여주고 오래 간직하게 해주는 기능에 돈을 받는다.

## 10. 구현 우선순위

### 1단계: 지도탭 발견 레이어

- 앱 진입 시 하단 `내 근처 이야기` 카드 덱 추가
- 좌우 슬라이드 가능
- 슬라이드 중 지도 마커는 움직이지 않음
- 기존 마커 선택 카드/확장 카드 흐름 유지

### 2단계: 스토리 남기기 명칭/문구 정리

- 하단 탭 `공유하기`를 `스토리 남기기`로 변경
- 작성 시트 타이틀/버튼/안내 문구 수정
- 사진 있음 = 스토리
- 영상 있음 = 숏츠 스토리
- 사진 없음 = 48시간 순간 기록

### 3단계: 내 지도 화면 재설계

- 기존 마이페이지를 내 지도 중심으로 변경
- 내 이미지/영상 마커 표시
- 카테고리 필터 추가
- 카테고리 색상 테두리 적용
- 내가 남긴 순간 리스트 진입점 추가

### 4단계: 숏츠 스토리 추가

- 영상 업로드/선택 흐름 추가
- 숏츠 썸네일 생성 또는 서버 썸네일 사용
- 영상 마커에 재생 아이콘/영상 배지 적용
- 지도탭 발견 카드에서 숏츠 스토리 표시
- 내 지도에서 숏츠 스토리도 카테고리 색상 테두리 적용

### 5단계: 카테고리 관리

- 카테고리 생성/수정
- 색상 선택
- 내 지도 필터와 연결

### 6단계: BM 재검토

제품 사용 흐름과 리텐션을 보고 BM을 다시 판단한다.

## 11. 디자인할 때 지켜야 할 기준

### 지도탭

- 검색보다 발견이 먼저 보여야 한다.
- 주변 이야기 카드가 첫 진입에서 보여야 한다.
- 마커는 너무 관리 도구처럼 보이면 안 된다.
- 순간 기록은 지도 위 생동감으로 활용한다.
- 숏츠 스토리는 영상임을 알 수 있어야 하지만, 지도탭을 숏폼 피드처럼 바꾸면 안 된다.

### 스토리 남기기

- SNS 공유가 아니라 장소 기록으로 느껴져야 한다.
- 사진/영상/텍스트가 모두 "이 장소에 남기는 기록"으로 묶여야 한다.
- 사진 없이 남기는 기록의 48시간 성격을 명확히 알려야 한다.
- 업로드 버튼도 `소식 업로드`보다 `스토리 남기기`가 좋다.

### 내 지도

- 지도 자체가 첫 화면의 주인공이어야 한다.
- 사진 스토리와 숏츠 스토리가 모두 내 기록 지도에 자연스럽게 쌓여야 한다.
- 카테고리 색상은 내 기록의 테마로 보여야 한다.
- 통계는 과하지 않게, 소유감을 돕는 정도로 둔다.
- 설정/공지/좋아요 목록은 하위 메뉴로 내려도 된다.

### 카테고리

- 제한하지 않는다.
- 색상으로 표현한다.
- My탭에서 강하게 드러낸다.
- 지도탭에서는 발견성을 해치지 않게 약하게 다룬다.
