# 마이페이지 · 나만의 지도 정보구조 (IA)

> 살아있는 문서. 최초 정리: 2026-06-29.
> 마이페이지를 지도 기반으로 "개편"할지 고민하다가, **마이페이지(허브)와 나만의 지도(콘텐츠 뷰)를 분리**하기로 결정한 기록.

## 결론 한 줄

마이페이지는 **기능 허브**로 그대로 유지한다. "나만의 지도"는 마이페이지를 대체하는 게 아니라, **별도의 콘텐츠 화면**으로 새로 추가한다.

## 왜 이렇게 정했나

마이페이지와 지도는 담는 내용의 성격이 다르다.

- **마이페이지 = 허브(기능 모음).** 설정, 공지사항, 좋아요한 스토리, 프로필 관리, 추후 팔로잉/팔로워 목록까지. 본질이 "목록·메뉴"라 리스트/카드가 맞는 그릇이다. 지도 위에 설정 버튼이나 팔로워 목록을 올리는 건 그릇이 안 맞는다.
- **나만의 지도 = 콘텐츠 뷰.** 내 스토리를 지도 위에서 보는 시각화 화면. 기능 허브가 아니라 "내 스토리의 공간적 표현"이다.

마이페이지를 통째로 지도화하면 허브 기능을 담을 곳이 사라진다. 그래서 둘을 분리한다.

## 두 화면의 책임

### 마이페이지 (허브) — owner 전용

리스트/카드 기반. 현재 구조를 거의 그대로 유지한다.

- 프로필 카드 (닉네임·소개·스토리 수)
- 내 스토리 (→ 나만의 지도 진입점, 아래 참조)
- 좋아요한 스토리
- 공지사항 · 설정
- (추후) 팔로잉 / 팔로워 목록

### 나만의 지도 (콘텐츠 뷰) — owner / visitor 모드

지도 기반. 같은 화면 하나에 `isOwner` 분기로 두 모드를 처리한다.

- 공통: 옅은 회색 지도 + 사진 마커(선 없음, 선택 시 강조) + 프로필 헤더 + 끌어올리는 시트
- **owner 뷰**: 비공개 스토리 포함, 설정 기어, 세그먼트(내 스토리 / 좋아요한)
- **visitor 뷰**: 공개 스토리만, 설정 기어 → 공유 + 팔로우, 시트는 "스토리 N개" 헤더
- 열 때 카메라는 내 스토리들의 bounds에 맞춘다(fitBounds) — "내 영역" 소유감의 핵심. 정적 시안엔 안 보이고 구현 단계에서 들어가는 동작.

owner/visitor 구분은 마이페이지가 아니라 **나만의 지도**에 속한다. 마이페이지는 본인만 보는 허브이고, 나만의 지도가 공유·방문이 일어나는 공개 표면이기 때문이다.

## 나만의 지도 진입점 (입구는 둘)

"나만의 지도"는 큰 기능으로 보고 진입로를 둘 다 연다. 화면 자체는 하나를 공유한다.

1. **마이페이지에서 진입.** 마이페이지의 "내 스토리" 영역을 **지도 미리보기 큰 버튼(map-card)** 으로 만들어, 탭하면 나만의 지도(전체화면)로 들어간다. 기존 가로 썸네일 스트립을 이 카드로 대체/보강한다.
2. **메인 '지도' 탭에서 진입.** 지도 탭 상단엔 이미 검색바 + 카테고리 칩 줄이 있다. 그 **검색바 우측의 프로필 아바타**를 탭하면 내 지도로 들어간다(지도·네이버 앱의 "내 공간" 진입 패턴). 아바타에 지도 핀 뱃지를 달아 "내 지도"임을 암시한다. 카테고리 칩 줄은 순수 필터로 유지한다.
   - 플로팅 토글("전체/내 지도")안은 상단 검색 영역과 충돌해서 폐기.

> 새 하단 탭은 추가하지 않는다. 기존 '지도' 탭과 "전체 지도 vs 내 지도" 역할이 겹쳐 혼란을 주고, 탭 추가 비용이 크기 때문이다.

## 방문자 경로 (추후 팔로우 기능)

팔로우한 친구 목록(마이페이지) → 친구 선택 → 그 친구의 **나만의 지도(visitor 뷰)**. 즉 나만의 지도가 곧 공개 프로필 표면이 된다.

## 무엇이 그대로고 무엇이 새로 생기나

- **그대로**: 마이페이지 허브(현재 카드 구조), 하단 탭 3개(지도/공유하기/마이페이지), 게시글 상세 시트·마커 자산.
- **새로**: 나만의 지도 화면(owner/visitor), 마이페이지 내 map-card 진입 버튼, 지도 탭의 "내 지도" 전환 버튼, bounds 카메라 동작.
