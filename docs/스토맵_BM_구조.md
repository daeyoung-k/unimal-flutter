# 스토맵(Stomap) BM & 기능 구조 정리

> "지도 위에 스토리를 남기자" — 1인 개발, 위치 기반 UGC 앱
> 작성일: 2026-06-15 · 상태: 방향 확정 / 디테일은 출시 후 데이터 보고 튜닝

---

## 1. 앱 정체성 방향

네이버·카카오·인스타와 **트래픽 발견 싸움(B 발견형)으로 정면 충돌하면 1인 개발은 진다.**
이길 수 있는 건 대기업이 안 하는 좁고 깊은 곳 — 두 축을 결합한다:

- **A. 기록형** — "내 발자취 / 추억 지도". 개인의 위치 기반 일기. → 감정적 lock-in, 리텐션.
- **C. 버티컬형** — 관심사 커뮤니티(런닝·낚시·등산·카페 등)가 지도 위에서 논다. → 좁지만 진한 결속.

**스윗스팟 = A + C.** "내 추억을 카테고리(테마)별로 쪼개 보는 나만의 지도."

---

## 2. BM 전체 구조 — 레이어링

BM은 하나로 가는 게 아니라 트래픽 피라미드에 층으로 쌓는다.

```
[탑]    로컬 제휴 / B2B   ← 트래픽 증거 쌓인 뒤 (나중, 영업 리소스 필요)
[미들]  구독             ← 핵심 팬, 높은 ARPU·리텐션  ★ 1인 개발 핵심 엔진
[베이스] 광고            ← 모든 무료 유저, 자동 베이스라인 (낮은 ARPU)
```

- **광고**는 DAU 안 받쳐주면 푼돈. "수익원"이 아니라 **프리미엄 전환 미끼**로 본다.
- **구독**이 1인 개발의 실질 먹거리. 메인 엔진.
- **로컬 제휴**는 위치 데이터를 가진 스토맵의 장기 무기지만, 영업 리소스 탓에 초기엔 옵션으로 보류.

## 3. 핵심 기능 3종 + 우선순위

### ⭐ 우선순위 1 — 내 스토리 지도 (무료, 전면 배치)

마이페이지 "내 스토리 전체보기"를 리스트가 아니라 **네이버 지도 위에 내 글 마커만 필터링**해서 표시.

- A(기록형)의 핵심. 다시 켜게 만드는 감정적 lock-in.
- "내가 이렇게 다녔구나" → **스크린샷 → SNS 공유 = 유저가 대신 해주는 공짜 홍보.**
- **절대 paywall 걸지 않는다.** 이게 리텐션·바이럴 엔진.
- 기존 지도 화면(`screens/map/`) 코드 재활용 가능.

### 우선순위 2 — 커스텀 카테고리 (A+C 결합, 부분 유료)

카테고리를 단순 폴더가 아니라 **내 추억 지도를 슬라이스하는 테마**로 만든다.

> 내 추억 지도에서 "등산"만 / "맛집"만 켜서 보기 → "내 ○○ 지도"를 여러 개 갖는 가치

- 1번(내 지도) 위에 얹어야 가치가 산다. 따로 팔면 "통행료" 느낌으로 반발.

### 우선순위 3 — 광고 (전환 미끼)

- **업로드 화면에만** 배너. 지도 화면은 절대 건드리지 않는다(정체성).
- 업로드는 저빈도 행동이라 노출량=수익은 적음 → 역할은 "살짝 거슬리게 해서 프리미엄으로 유도".

## 4. 카테고리 구조 (확정 모델)

### 온보딩 — "골라 담기"

미리 다 깔지 않고(빈 슬롯 낭비), 완전 빈 값도 아니고(empty state 이탈),
**추천 카테고리를 칩으로 보여주고 유저가 직접 고르게** 한다.

```
첫 진입 → "어떤 스토리를 남길 거예요?"
[🏃 런닝] [🎣 낚시] [⛰️ 등산] [☕ 카페] [🍜 맛집] [✈️ 여행] ...
→ 탭해서 선택 + 직접 추가도 가능
```

- 빈 화면 문제 해결, 기능 학습(튜토리얼 불필요), 자기가 고른 것 = 소속감↑.
- 결제벽을 **유저가 "하나 더!"를 원하는 순간** 만나게 됨 (전환율 최적 타이밍).

### 무료 한도 — 모델 A (보유 제한)

| | 설명 |
|---|---|
| **채택** | 무료는 카테고리를 **총 N개까지 보유** (디폴트/커스텀 구분 없이 합산) |
| **결제 트리거** | N+1번째를 **만들려는 순간** → 프리미엄 유도 |
| **핵심** | "보유 = 곧 사용." 잠긴 칸·활성/비활성 토글 같은 개념 **없음** |

**❌ 모델 B (무제한 생성 + 3개만 활성화)는 채택 안 함** — "만들었는데 왜 못 써?" 반발 + UI 상태 복잡(오버엔지니어링).

- 멘탈 모델 단순: "무료는 N개, 더 필요하면 프리미엄."
- 데이터 잠금이 아니라 **신규 생성만 막음** → 거부감 적음.
- 개수 체크는 **생성 시점 한 줄**로 끝남.

### 한도 값(N) — 일단 3, 추후 튜닝

- 3개는 전환은 잘 되나 "인색하다"는 첫인상 위험(런닝+맛집+여행이면 벌써 꽉 참).
- **3~5 사이에서 출시 후 사용 패턴 보고 조정.** 값은 **서버 설정값으로 빼두기** → 앱 수정 없이 변경.

## 5. 구독 구조 — 기능별로 쪼개 팔지 않는다

결제 포인트가 흩어지면 유저도 헷갈리고 관리도 지옥. **하나의 티어로 번들.**

```
스토맵 프리미엄  (월 ₩3,900 / 연 ₩29,000 가량 — 가격은 추후 확정)
 ├ 커스텀 카테고리 무제한      (무료는 N개)
 ├ 광고 제거                   ← 단독으론 약한 셀링포인트, 번들 보너스로
 ├ 프리미엄 마커 / 테마
 └ (나중) 지도 백업·내보내기
```

- "광고 제거를 따로 팔까?" 고민 해소 → 프리미엄에 끼워주는 perk로 정리.
- 연 구독 옵션 제공으로 전환·리텐션 보강.

## 6. 내 스토리 지도 — 배치 & 구현 구조

### 두 진입점 (역할 분리)

| 진입점 | 역할 | 동작 |
|---|---|---|
| **메인 지도 토글** | 빠른 유틸 ("내 글 어디 있더라") | 지도 위에서 데이터만 내 글로 교체 + 줌 빼서 auto-fit |
| **내 지도 탭** | 감정적 공간 + 공유 | 통계·카테고리 필터·공유가 붙는 풀 경험 |

### 핵심 원칙 — 화면 ≠ 재사용, 지도 위젯만 재사용

```
        StoryMapView (재사용 위젯)        ← 네이버 지도 + 마커 렌더링만
        입력: posts, onTap, bounds          "내 글/남 글" 개념 없음
          ▲ 사용              ▲ 사용
   MapScreen (메인)      MyStoryMapScreen
   데이터: 근처 인기글     데이터: /board/my/posts
   UI: 검색·발견필터       UI: 프로필·통계·공유·카테고리 칩
```

- **화면은 따로**, 그 안의 지도 위젯만 공유. 플래그(`isMyMap`)로 한 화면 분기 ❌ → 조건문 번식 = 유지보수 지옥.
- 위젯 책임: 지도 초기화, 마커 렌더링(`globalZIndex` 포함), 탭 콜백, 카메라/auto-fit.
- 화면 책임: 데이터 fetch, 상·하단 UI, 앱바, empty state.
- 첫 작업 = 기존 `screens/map/`에서 순수 지도 부분을 `StoryMapView`로 추출하는 리팩터링.

### ⚠️ 백엔드 — 전용 엔드포인트 필요

기존 `GET /board/map/location/post`는 **뷰포트+score 기반(근처 인기글)** → "위치 무관 내 글 전부"가 안 잡힘.

```
GET /board/my/posts   ← 내 글 전체 (좌표·카테고리·썸네일 포함, 뷰포트 무관)
```

- 메인 토글 / 내 지도 둘 다 **이 엔드포인트 하나** 공유. 카테고리 필터는 응답 `category` 필드로 클라에서 거름.

### 마이페이지 = 지도 hero

지도를 바닥에 깔고 프로필·통계·설정은 오버레이/바텀시트로. 설정은 우상단 ⚙️ 아이콘 뒤로.

### ⚠️ empty state (홍보 전 = 글 0~몇 개 유저 다수)

- 0개: "첫 스토리를 남겨보세요" + 작성 유도. 1~2개: 격려 문구. → 안 챙기면 핵심 기능이 역효과.

## 7. 네비게이션 구조 (4탭)

```
[🗺️ 홈]   [🔍 탐색]    [ ＋ ]     [📍 내 지도]
 현재지도    발견탭       남기기      마이페이지
            (리스트)     (강조)      리뉴얼
```

- **홈**: 현재 지도 화면 (그대로).
- **탐색**: 바텀카드 발견탭 — "내 주변 글" / "핫한 글" 리스트. ⚠️ **빈 탭으로 출시 금지** — 최소 버전이라도 돌게 하거나, 없으면 3탭으로 시작 후 추가.
- **＋ 남기기**: 글 작성. 가운데 **시각적으로 강조**(원형 ＋). ⚠️ "공유하기"라 부르지 말 것 — SNS 공유 기능과 의미 충돌.
- **내 지도**: 마이페이지 리뉴얼. 아이콘도 person → 핀 계열로 교체.

## 8. 구현 순서 (로드맵)

1. **내 스토리 지도** — 무료, 전면. 마이페이지 전체보기에 네이버 지도 + 내 글 마커 필터.
2. **카테고리 + 온보딩 골라담기** — 그 지도 위 필터로 통합. 보유 N개(모델 A).
3. **프리미엄 티어 하나** — 카테고리 무제한 + 광고 제거 + 마커 번들. 결제 인프라 구축.
4. **광고** — 업로드 화면 배너만, 프리미엄 전환 미끼.

## 9. 미정 / 추후 데이터 보고 결정

- 무료 카테고리 한도 N (3 vs 5) — 서버 설정값, 출시 후 튜닝.
- 프리미엄 가격(월/연), 정확한 기능 라인업.
- 디폴트(프리셋) 카테고리 정의를 **서버에 둘지 / 앱에 둘지** — 다음 설계 안건.
- 로컬 제휴 진입 시점 (트래픽 증거 확보 후).

### Phase 3 — 소셜 / 크리에이터 BM (나중, 신중히)

- 팔로잉/팔로워 + 공개 visibility 레이어는 **네트워크가 돌아갈 때** 의미. 0 유저 단계엔 부적합.
- ❌ **"공개 카테고리 수 제한"식 과금은 채택 안 함** — 공유=성장 엔진인데 그걸 막으면 성장을 직접 깎음(거꾸로 된 인센티브). + 단지 이 paywall 하나로 소셜 그래프 전체를 짓는 건 배보다 배꼽.
- ⭕ 방향은 **막지 말고 보상** — 크리에이터 PRO 도구에 과금: 스토리 통계(조회수·인기 지역), 공개 프로필/지도 커스텀, 상단 노출, 인증 배지. 공유를 **늘리는** 쪽이라 성장과 같이 감.
- 정리: 소셜 BM = Phase 3, 레버는 "visibility 제한"이 아니라 "크리에이터 PRO".

> ⚠️ 현실 체크: 아직 홍보 전이라 **유저 수요 데이터가 없음.**
> 지금 paywall을 완벽히 깎는 건 오버엔지니어링.
> **2번(내 지도)을 빨리 내고 → 사람 모이는지·뭘 더 원하는지 보고 → 프리미엄 디테일 확정.**
> 결제 인프라는 미리 깔되, 가격·기능은 반응 보고 조정.
