# 텍스트 마커 로컬 전환 복구 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 같은 API 줌 단계에서도 iOS 텍스트 마커가 현재 줌의 점·말풍선 표현과 항상 일치하도록 API 없는 로컬 전환을 복구한다.

**Architecture:** 줌 히스테리시스는 작은 순수 정책으로 분리하고, 메인 지도는 현재 `_postGroups`와 등록된 마커만 사용해 표현을 in-place로 동기화한다. 비동기 아이콘 생성은 표현 세대 번호로 latest-wins를 보장하며, API 응답 적용 후에도 `nowCameraPosition` 기준으로 한 번 더 동기화한다.

**Tech Stack:** Flutter, Dart, flutter_naver_map 1.4.4, flutter_test

## Global Constraints

- 점 -> 말풍선 진입 줌은 `17.3`, 말풍선 -> 점 복귀 줌은 `16.8` 미만이다.
- 로컬 표현 동기화는 `_loadMapMarkers`, `addOverlay`, `deleteOverlay`를 호출하지 않는다.
- 기존 165ms 페이드인은 유지하고 크로스페이드는 구현하지 않는다.
- 사진 마커, 스택 마커, 백엔드 API, 내 지도 화면은 변경하지 않는다.
- 오래된 비동기 아이콘 결과는 적용하지 않고 현재 마커를 유지한다.
- iOS를 우선 검증 플랫폼으로 사용한다.
- 계획과 기능 변경은 사용자 검토 전 커밋하지 않는다. 사용자가 `커밋해줘`라고 명시한 뒤에만 커밋한다.

- [ ] **Step 4: Extract one shared eligibility helper**

Move the current screen-dp density calculation out of `_loadMapMarkersInternal()` into:

```dart
Set<String> _textBubbleEligibleIds({
  required List<List<MapPost>> markerGroups,
  required double latitude,
  required double rawZoom,
}) {
  final meterPerDp = _mapController!.getMeterPerDpAtLatitude(
    latitude: latitude,
    zoom: rawZoom,
  );
  final denseRadiusDeg =
      kTextCardDenseRadiusDp * meterPerDp / 111320.0;
  final eligible = <String>{};

  for (final group in markerGroups) {
    if (group.length != 1) continue;
    final top = group.first;
    if (top.fileInfoList.isNotEmpty) continue;

    var neighbors = 0;
    for (final other in markerGroups) {
      final candidate = other.first;
      if (identical(candidate, top)) continue;
      if ((candidate.latitude - top.latitude).abs() < denseRadiusDeg &&
          (candidate.longitude - top.longitude).abs() < denseRadiusDeg) {
        neighbors++;
        if (neighbors >= kTextCardDenseNeighbors) break;
      }
    }
    if (neighbors < kTextCardDenseNeighbors) eligible.add(top.id);
  }
  return eligible;
}
```

Use this helper from both `_loadMapMarkersInternal()` and local synchronization so density rules cannot diverge.

- [ ] **Step 5: Add latest-wins local synchronization**

Add state:

Implement a helper with this behavior:

```dart
Future<void> _syncTextMarkerPresentationForZoom({
  required double rawZoom,
  required double latitude,
}) async {
  if (!mounted || _mapController == null) return;
  final generation = ++_textPresentationGeneration;
  final globalCardMode = _resolveTextCardMode(rawZoom);
  final groups = List<List<MapPost>>.from(_postGroups);
  final eligibleIds = _textBubbleEligibleIds(
    markerGroups: groups,
    latitude: latitude,
    rawZoom: rawZoom,
  );

  if (_selectedGroupIndex != null) {
    if (globalCardMode) _textBubbleRestoreIds.addAll(eligibleIds);
    return;
  }

  final colors = AppColors.of(context);
  for (final group in groups) {
    if (!mounted || generation != _textPresentationGeneration) return;
    final post = group.first;
    final toCard = globalCardMode && eligibleIds.contains(post.id);
    if ((_textMarkerCardMode[post.id] ?? false) == toCard) continue;
    await _flipTextMarkerModeInPlace(
      post,
      toCard: toCard,
      tierSize: _markerBaseSize[post.id] ?? _normalMarkerSize,
      isHot: _markerIsHot[post.id] ?? false,
      colors: colors,
      expectedPresentationGeneration: generation,
    );
  }
}
```

Restrict the loop to registered text markers by requiring `_markerRefs[post.id] != null` and `post.fileInfoList.isEmpty` before calling the flip.
