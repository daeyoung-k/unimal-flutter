# 텍스트 마커 리디자인 — 구현 계획 (2026-07-14)

## 배경

기존 텍스트 마커(파란 말풍선 + 가짜 줄 3개)가 날카롭고 콘텐츠가 없다는 피드백.
당근 스타일을 참고해 **화이트 콘텐츠 카드 + 점 앵커** 구조로 리디자인 확정.
디자인은 Figma에서 확정 완료:

- 스펙 시트: `18 텍스트 마커 변형 시트` — https://www.figma.com/design/zL0UifobuW5936gNE5NmZq/작업영역?node-id=239-36
- 줌 시나리오(4패널): `18-2 텍스트 마커 — 줌 시나리오` — https://www.figma.com/design/zL0UifobuW5936gNE5NmZq/작업영역?node-id=245-34

## 확정 스펙

### 말풍선 (줌인 카드, 꼬리 없음)
- surface 흰색 · border 1px(`border` 토큰) · radius 14 · 그림자 y2 blur8 8%
- 제목: Pretendard SemiBold 13 / `textPrimary` · 시간: Regular 10 / `textMuted`
- 본문: Regular 12 / `textSecondary` · 최대 2줄 말줄임 · 본문 폭 180 고정
- **꼬리 없음** — 점 마커가 좌표 앵커 역할 (말풍선은 점 위 4px 간격)

### 점 마커 (줌아웃, 항상 좌표 앵커)
- 32dp 화이트 원 + border 1px + `primaryStrong` 챗 글리프 + 하단 꼬리 다이아 9dp
- 스택: 우상단 +N 뱃지 — 16dp `primary` 원 + 화이트 링 1.5 + SemiBold 8.5 화이트
- 사진 마커(원형 링)와 같은 패밀리 유지

### 노출 규칙 (기존 점↔카드 히스테리시스 재사용 — `_textCardMode` 로직 유지)
- 줌 < 17.3: 점 + 캡션만
- 줌 ≥ 17.3 진입 / 16.8 미만 이탈 (히스테리시스): 단일 + 비밀집 → 말풍선 노출
- 스택(2+): 항상 점+N (탭 → 원형 펼침) · 밀집(50m 내 그룹 2+): 말풍선 억제 (C안)
- **바텀 카드 열림: 모든 말풍선 숨김, 점만 표시** — 선택 마커는 z부스트 + 1.18x
- 탭: 상태 무관 줌인 + 하단 카드

### 캡션
- 기본 8자 말줄임 (`kMarkerCaptionMaxChars` 5 → 8)
- 선택(포커스) 마커: 전체 타이틀 — `NOverlayCaption.requestedWidth`로 2~3줄 랩
  + `setIsForceShowCaption(true)` 충돌 보호. 해제 시 8자 캡션 복원
- 구현 위치: `_applySelectionHighlight` (setCaption은 네이티브 setter — 비트맵 재생성 없음)

## 구현 순서

1. **점 마커 비트맵** — `marker_image_factory.dart` `createTextDotImage()` 교체
   (화이트 원 + 챗 글리프 + 꼬리, 스택 뱃지는 기존 `addClusterBadge` 파이프라인 호환 200x200 유지)
2. **말풍선 위젯 리스타일** — `text_marker_widgets.dart` `TextMarkerCard`
   (꼬리 제거, 화이트 카드 스펙 적용, 점과 합성 — 카드+점을 한 비트맵/위젯으로, anchor 0.5/1.0 유지)
3. **카드 열림 시 말풍선 숨김** — `map_naver.dart` (선택/카드 상태에서 텍스트 카드 마커 → 점 표현 강제)
4. **캡션 8자 + 선택 시 전체 표시** — `marker_constants.dart` + `_applySelectionHighlight`

각 단계 후 `flutter analyze` 통과 확인. 내지도(`my_story_map_screen.dart`)도 같은 마커 시스템 공유 — 상수/팩토리 변경 영향 확인할 것.
