# 나만의 지도 구현 계획 — 어떻게 / 어떤 단계로

> 설계(무엇을/왜): `docs/specs/2026-06-29-나만의지도.md`
> IA 방침: `docs/architecture/마이페이지-나만의지도-정보구조.md`
> 작성: 2026-06-29. 단계는 체크박스(`- [ ]`)로 추적.

**Goal:** "나만의 지도" 화면(owner/visitor)을 신설하고, 마이페이지·지도 탭 두 곳에서 진입을 연결한다. 마이페이지 허브는 유지.

**Architecture:** `screens/map/`의 기존 지도/바텀시트/마커 자산을 재사용하는 신규 `MyMapScreen(isOwner, memberId?)`. 마커는 라이브러리 `[지도] 커스텀 마커` 톤, 선택 강조는 `marker-as-strip` 패턴. 마커 탭 → 기존 `PostDetailSheet`. 진입 B는 `profile.dart`의 "내 스토리" 영역을 map-card 버튼으로 교체, 진입 A는 지도 탭 상단 검색바 우측 아바타 버튼 추가.

**Tech Stack:** Flutter, GetX, flutter_naver_map, AppColors 토큰, BoardApiService, MapCardExpandedContent/PostDetailSheet.

| 파일 | 작업 |

| `lib/screens/map/my_map/my_map_sheet.dart` | 신규: 하단 시트(owner 세그먼트 / visitor 헤더) + 리스트 |
| `lib/service/board/board_api_service.dart` | 내 스토리/특정 멤버 공개 스토리 위치 조회 추가 |
| `lib/screens/profile/profile.dart` | "내 스토리" 스트립 → map-card 진입 버튼으로 교체 |

> 정확한 지도 탭 파일명/구조는 착수 시 `screens/map/` 확인 후 확정.

## Phase 0 — 백엔드 확인 (착수 전 결정, blocking)

- [ ] **내 스토리 위치 조회**: 기존 `GET /board/map/location/post`(전체, score 정렬)에 작성자 필터를 더할지, 신규 엔드포인트(`/board/map/location/my` 등)를 팔지 결정.
- [ ] **visitor용**: 특정 `memberId`의 **공개** 스토리만 위치 조회 가능한지. 공개/비공개 구분 필드 존재 여부 확인.
- [ ] bounds 계산은 클라(반환된 좌표 min/max)에서 처리 — 서버 변경 불필요.

> 권대영(백엔드)이 직접 판단 가능한 영역. visitor/팔로우는 추후라 1차 구현은 owner 경로만 있어도 된다.

## Task 1 — MyMapScreen 골격 (owner 우선)

**Files:** Create `lib/screens/map/my_map/my_map_screen.dart`

- [ ] `MyMapScreen({required bool isOwner, String? memberId})` 정의. GetX 라우트 인자로 주입.
- [ ] 기존 지도 화면(`screens/map/`)의 네이버 지도 위젯/마커 로딩 패턴을 참고해 베이스 지도 구성.
- [ ] 내 스토리 마커 데이터 로드 (Phase 0 API). owner: 내 전체, visitor: memberId 공개분.
- [ ] 배경/색은 `AppColors.of(context)` 토큰만 사용 (하드코딩 금지 — CLAUDE.md 규칙).
- [ ] `flutter analyze` 통과.

## Task 2 — 마커 + bounds 카메라

- [ ] 사진 썸네일 마커 렌더 (첫 이미지 썸네일). 동선 polyline 없음.
- [ ] 선택 시 파란 링 강조 — `ImageService.addSelectionRing`(`marker-as-strip`) 재사용.
- [ ] **진입 시 fitBounds**: 마커 좌표 min/max로 카메라 영역 맞춤 (스토리 0~1개일 때 fallback 줌 처리).
- [ ] 마커 탭 → `showPostDetailSheet(context, boardId)` 재사용.

## Task 3 — 프로필 헤더 + 상단 액션

**Files:** Create `my_map_profile_header.dart`

- [ ] 좌상단 프로필 칩: 아바타 + 닉네임 + 스토리 수 (지도 위 floating).
- [ ] `isOwner` 분기:
  - owner → 우상단 설정 기어 (마이페이지 설정 시트/화면 연결).
  - visitor → 우상단 공유 + 팔로우 버튼 (팔로우는 추후 stub).

## Task 4 — 하단 시트 (owner 세그먼트 / visitor 헤더)

**Files:** Create `my_map_sheet.dart`

- [ ] 끌어올리는 `DraggableScrollableSheet`.
- [ ] owner: 세그먼트 토글 `내 스토리 N / 좋아요한 N`. 좋아요한은 `getMyLikedPostList`(기존) 재사용.
- [ ] visitor: `스토리 N개` 헤더 (세그먼트 없음).
- [ ] 리스트 행: 썸네일 + 제목 + 위치 + 시간 + 좋아요. `story_list.dart`/`StoryThumbnailCard` 패턴 재사용.
- [ ] 행 탭 → `PostDetailSheet`.

## Task 5 — 진입 B · 마이페이지 map-card (확정 디자인)

**Files:** Create `my_map_entry_card.dart`, Modify `profile.dart`

- [ ] `profile.dart`의 기존 "내 스토리" 가로 스트립 섹션을 **map-card 버튼**으로 교체.
- [ ] map-card: 미니 지도(내 마커 미리보기) + `내 지도 보기` + `N개의 스토리 · 지역` + 화살표.
  - 미리보기 지도는 정적 스냅샷 또는 경량 지도. 비용 크면 1차는 대표 썸네일 콜라주 + 핀 오버레이로 대체 가능.
- [ ] 탭 → `Get.toNamed('/my-map', arguments: {isOwner:true})`.

## Task 6 — 진입 A · 지도 탭 아바타 진입

**Files:** Modify 지도 탭 상단 위젯

- [ ] 검색바 우측에 프로필 아바타 버튼(지도 핀 뱃지) 추가.
- [ ] 탭 → `Get.toNamed('/my-map', arguments: {isOwner:true})`.
- [ ] 카테고리 칩 줄은 건드리지 않음 (필터 그대로).

## Task 7 — 라우트 + 마무리

- [ ] `app_routes.dart`에 `/my-map` 등록.
