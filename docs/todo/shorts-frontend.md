# 숏츠 기능 개발 예정 - Frontend

작성일: 2026-06-17  
상태: 개발 예정  
범위: Flutter 프론트 화면/컴포넌트

---

## 목표

숏츠는 별도 SNS 피드가 아니라 **장소에 남기는 짧은 영상 기록**이다.

사진 스토리처럼 지도와 내 지도에 남되, 영상 콘텐츠임을 한눈에 알 수 있어야 한다.

---

## 주요 작업

### 1. 영상 업로드

- 스토리 남기기 시트에 영상 추가 기능 추가
- 사진과 영상 업로드 UX 구분
- 영상 선택 후 썸네일 미리보기 표시
- 영상 길이 제한 적용

### 2. 영상 길이 제한

초기 기준은 짧게 가져간다.

후보:

- 15초
- 30초

1차 추천:

- 최대 30초

긴 영상 편집 앱이 아니라 장소의 짧은 분위기를 남기는 기능으로 유지한다.

### 3. 숏츠 전용 마커

- 영상 썸네일 기반 마커 사용
- 기존 이미지 마커와 형태는 최대한 맞춘다.
- 영상임을 알 수 있도록 작은 재생 아이콘 또는 영상 배지 추가
- My탭 내 지도에서는 카테고리 색상 테두리 적용
- 지도탭에서는 과하게 튀지 않도록 발견성을 우선

### 4. 숏츠 카드

마커 선택 시 기존 카드 구조를 유지하되 영상 콘텐츠임을 표시한다.

- 썸네일 영역에 재생 아이콘 표시
- 카드에서 영상 자동 재생은 1차 보류
- 탭하면 확장 카드에서 재생하는 방향 검토

### 5. 확장 카드

- 확장 카드 상단 미디어 영역에서 영상 재생
- 재생/일시정지 컨트롤 필요
- 음소거 기본값 검토
- 댓글/좋아요 구조는 기존 스토리와 동일하게 유지

### 6. 내 지도 반영

- 내 지도에서 숏츠 스토리도 마커로 표시
- 사진 스토리와 동일하게 카테고리 필터 대상
- 숏츠 마커에는 영상 배지 표시
- 카테고리 색상 테두리 적용

---

## 보류 결정

- 영상 최대 길이 최종값
- 자동 재생 여부
- 기본 음소거 여부
- 영상 압축/업로드 진행률 UI
- 서버 썸네일 사용 vs 클라이언트 썸네일 생성
- 사진과 영상을 한 게시글에 같이 허용할지 여부

---

## 디자인 기준

- 숏츠는 지도 위에서 튀는 광고처럼 보이면 안 된다.
- 사진 스토리와 같은 가족처럼 보이되, 영상임은 즉시 알아야 한다.
- 스토맵의 핵심은 숏폼 피드가 아니라 장소 기록이다.
