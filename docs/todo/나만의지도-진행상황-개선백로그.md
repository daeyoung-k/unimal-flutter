# 나만의 지도 — 진행 상황 / 개선 백로그

> 최종 정리: 2026-06-30
> 관련 문서: `docs/specs/2026-06-29-나만의지도.md` (무엇을/왜), `docs/plans/2026-06-29-나만의지도.md` (단계),
> `docs/architecture/마이페이지-나만의지도-정보구조.md` (IA)

지도 탭과 마이페이지 두 곳에서 진입하는 **내 스토리 전용 지도("나만의 지도", owner)** 작업 기록과 다음 개선 과제.

---

## 1. 지금까지 구현 (완료)

### 진입점
- **지도 탭**: 우측 하단(네비게이션 바 위)에 핀 아이콘 + "내 지도" 알약 버튼(`_buildMyMapButton`).
  탭 → `/my-story-map`. 카드 열리면 페이드아웃. 검색바는 풀폭 유지.
  - 좌측 "내 위치" 버튼과 높이(36)·`bottom:45` 정렬.
- **마이페이지**: 기존 "내 스토리" 썸네일 스트립 → 라이브 미니맵 카드(`MyStoryMapCard`)로 교체. 탭 → `/my-story-map`.

### 전용 화면 (`lib/screens/profile/mypage/my_story_map_screen.dart`)
- **프로필 헤더**: 좌상단 뒤로가기 + 프로필 칩(아바타 + 닉네임). (설정 기어·스토리 수는 제거 — 설정은 마이페이지로 일원화)
- **커스텀 마커**: 메인 지도와 동일 파이프라인. 사진 글 = 썸네일(`ImageService.createMarkerImage`),
  텍스트 글 = 점(`createTextDotImage`), 32px + 제목 캡션. 탭 → 상세 시트(`showPostDetailSheet`).
- **클러스터링 (B안: 묶음 + 탭 펼침)**:
  - `NClusterableMarker` + `clusterOptions`. 줌 0~15 묶음, 16+ 개별 펼침.
  - 클러스터 = 카운트 버블(`_ClusterBubble`, primary 원 + 화이트 링 + 숫자) — "이 지역에 N개" 표현.
  - 클러스터 탭 → 해당 위치로 줌인. 단일/펼친 마커 탭 → 상세 시트.
- **하단 바텀시트**: `DraggableScrollableSheet`(초기 0.32 / 0.14~0.9 snap). 드래그 핸들·세그먼트·리스트를
  한 ListView 안에 둬서 어디를 잡아도 끌림. 세그먼트 토글 `내 스토리 N / 좋아요한 N`(좋아요한은 지연 로드).
  리스트 행(썸네일+제목+위치+시간+좋아요/댓글) 탭 → 상세 시트.
- **마커 렌더 타이밍 픽스**: `_onMapReady`(지도 준비)와 `_load`(데이터) 중 늦게 끝나는 쪽에서 `_renderMarkers()`
  1회 실행(`_markersRendered` 가드) — 비동기 순서 무관하게 마커 표시.
- **좌표 필터**: 서버가 위치 없는 글에 `(0,0)`을 주므로 `_located`에서 제외.

### 메인 지도
- 확장 카드 높이를 내 지도 시트와 동일 기조로 맞춤 (`minTopMargin = 화면높이*0.1(사용자 조정값) + 네비바 소비높이`).
  네비게이션 바(높이 64 + 하단 safe area)만큼 본문이 줄어든 좌표 차이를 보정.

## 2. 앞으로 개선할 것 (백로그)

### (1) 첫 진입 줌 레벨 정하기 — ⚠ 우선
**증상**: 진입 직후 줌이 애매해서 클러스터가 N개로 쪼개졌다가 하나로 합쳐졌다 하는 깜빡임이 보임.

**원인 추정**:
- `_fitCamera`의 진입 줌(`_maxFitZoom = 13.0`)이 `clusterOptions.mergeStrategy`의 병합 거리 경계
  (줌 13~14 = 80dp 등)와 겹쳐, 카메라 미세 이동에도 묶임/풀림이 토글됨.
- 메인 지도의 텍스트 마커처럼 **히스테리시스(enter/exit 줌 분리)** 가 없음.

**해결 방향(택1 또는 조합)**:
- 진입 줌을 병합이 안정적인 값으로 고정(13 ↔ 다른 값 실험), `fitBounds` 결과를 병합 경계에서 떨어뜨리기.
- `mergeStrategy`의 줌 구간/거리 재조정.
- 진입은 `scrollAndZoomTo`로 단일 고정 줌만 쓰고 fitBounds 후처리를 줄이기.
- 손볼 곳: `_fitCamera`, `_maxFitZoom`, `NaverMap.clusterOptions.mergeStrategy`.

### (2) 클러스터 탭 줌인 레벨 + 동일 좌표 펼침 — ⚠ 우선
**증상**: 클러스터 탭 시 줌인은 되지만, **같은 주소(동일 좌표)에 뭉친 글**은 16+로 펼쳐도 좌표가 같아
전부 겹쳐서 몇 개인지·어떤 글인지 구분 불가. (이전에 시도한 꽃잎 펼침을 B안 전환하며 제거함)

**해결 방향(택1 또는 조합)**:
- **동일 좌표군 한정 펼침(spiderfy)**: 클러스터 탭이 "좌표가 거의 같은 글들"이면 줌인 대신
  중심 주변 원으로 펼쳐 표시(이전 `_fanOutPositions` 로직 재활용). 좌표가 흩어진 클러스터면 기존처럼 줌인.
- **클러스터 탭 → 그 위치 글만 하단 시트로**: 펼침 대신 바텀시트에 해당 위치 글 리스트.
- 클러스터 탭 목표 줌을 데이터 분포에 맞게(동일점이면 펼침, 분산이면 16~17) 분기.
- 손볼 곳: `_buildClusterMarker`의 클러스터 탭 핸들러, (재도입 시) `_fanOutPositions`.
