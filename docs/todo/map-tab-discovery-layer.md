# 지도탭 발견 레이어 디자인 예정

작성일: 2026-06-17  
상태: 디자인/구현 예정  
관련 문서: `docs/main/스토맵 방향성 구조 설계.md`

---

## 1. 목표

지도탭은 스토맵에 들어왔을 때 가장 먼저 만나는 화면이다.

이 화면은 단순히 지도를 보여주는 곳이 아니라, 사용자가 바로 이런 감정을 느끼게 해야 한다.

> "내 주변에 사람들이 남긴 이야기가 있네."  
> "지도 위에 지금의 기록이 떠다니고 있네."  
> "가까운 곳부터 하나씩 넘겨보고 싶다."

따라서 1차 목표는 **지도탭 첫 진입에서 발견의 감정**을 강화하는 것이다.

---

## 2. 지도탭이 가져야 할 감정

### 발견

사용자는 앱을 켜자마자 주변 이야기를 훑을 수 있어야 한다.

- 내 주변 글
- 인기 글
- 최신 글
- 순간 기록

이 요소들은 "게시판 필터"처럼 보이면 안 되고, 지도 위 이야기를 탐색하는 모드처럼 보여야 한다.

### 가벼운 탐색

처음부터 상세 카드가 너무 무겁게 올라오면 지도 탐색이 막힌다.

첫 진입에서는 작고 가벼운 카드 영역으로 주변 글을 넘겨보게 한다.

### 지도 위 생동감

텍스트-only 48시간 기록은 지도탭의 생동감을 만든다.

사진 스토리는 오래 남는 기록이고, 순간 기록은 지금 이 장소에 붙은 짧은 말이다.

---

## 3. 화면 구조

### 상단 영역

기존 검색바는 유지한다.

역할:

- 장소 검색
- 사용자가 특정 장소를 찾고 싶을 때 사용

주의:

- 검색바가 화면의 주인공처럼 보이면 안 된다.
- 검색보다 발견 카드가 먼저 눈에 들어오도록 하단 발견 영역을 강화한다.

### 발견 모드 버튼

검색바 아래 또는 지도 상단의 안전한 위치에 작은 칩 버튼으로 배치한다.

1차 버튼:

- 내 근처 글
- 인기 글
- 최신 글

추가 후보:

- 순간 기록

초기 선택값:

- `내 근처 글`

디자인 방향:

- 둥근 칩 형태
- 선택된 칩은 스토맵 블루 계열로 강조
- 비선택 칩은 흰색/반투명 배경
- 지도 POI를 너무 많이 가리지 않도록 높이는 작게 유지

권장 배치:

```text
[검색바]
[내 근처 글] [인기 글] [최신 글]
```

### 지도 마커 영역

기존 마커 구조는 유지한다.

- 사진 있는 스토리: 기존 이미지 마커
- 텍스트-only 순간 기록: 기존 텍스트 말풍선/텍스트 마커

중요:

- 발견 카드 슬라이드 중에는 마커가 따라 움직이지 않는다.
- 사용자가 마커를 직접 선택했을 때만 기존 선택 카드 흐름으로 들어간다.

---

## 4. 첫 진입 카드 영역

### 이름 후보

첫 진입 카드 영역의 이름은 너무 서비스 설명처럼 길지 않게 한다.

후보:

- 내 근처 이야기
- 주변 이야기
- 지금 근처 이야기

추천:

> 내 근처 이야기

이유:

- "글"보다 "이야기"가 스토맵답다.
- 사용자 위치 기준의 발견 감정이 있다.
- 너무 기능명처럼 딱딱하지 않다.

### 위치

하단 바텀 내비 위에 작게 떠 있는 카드 덱으로 배치한다.

권장 구조:

```text
지도

        마커들

┌────────────────────────┐
│ 내 근처 이야기          │
│ [카드] [카드] [카드] →  │
└────────────────────────┘

[지도] [스토리 남기기] [내 지도]
```

### 카드 높이

처음 보이는 카드 영역은 기존 마커 선택 카드보다 낮고 가벼워야 한다.

권장:

- 지도 화면의 약 18~24%
- 바텀 내비와 겹치지 않음
- 한 손으로 좌우 스와이프하기 편한 높이

### 카드 구성

카드는 주변 글을 빠르게 훑는 용도다.

사진 스토리 카드:

- 썸네일
- 제목 또는 본문 첫 줄
- 장소명/동네명
- 시간
- 좋아요/댓글 수는 작게

텍스트 순간 기록 카드:

- 텍스트 말풍선 느낌
- 48시간 기록임을 작게 표시
- 장소명/동네명
- 시간

카드에서 과하게 보여주지 않을 것:

- 긴 본문 전체
- 댓글 미리보기
- 큰 액션 버튼
- 작성자 정보 과다 노출

### 카드 클릭 동작

카드를 탭하면 기존 마커 선택 카드 흐름으로 진입한다.

동작:

1. 사용자가 `내 근처 이야기` 카드를 좌우로 넘긴다.
2. 이때 지도 카메라와 마커는 움직이지 않는다.
3. 사용자가 특정 카드를 탭한다.
4. 해당 스토리의 기존 선택 카드가 열린다.
5. 필요하면 해당 마커를 선택 상태로 강조한다.

---

## 5. 버튼별 동작

### 내 근처 글

현재 지도 중심 또는 사용자 현재 위치 기준으로 가까운 스토리를 보여준다.

노출 콘텐츠:

- 사진 스토리
- 숏츠 스토리
- 텍스트 순간 기록

내 근처 글은 "주변에 뭐가 있지?"가 핵심이므로 콘텐츠 타입을 가리지 않는다.

정렬 기준 후보:

- 가까운 순
- 최근성 + 거리 혼합

디자인 감정:

> "지금 내 주변에 뭐가 있지?"

### 인기 글

지도 범위 또는 주변 범위 안에서 반응이 좋은 스토리를 보여준다.

노출 콘텐츠:

- 사진 스토리
- 숏츠 스토리

인기 글은 사용자가 눌렀을 때 볼거리가 있어야 하므로 이미지/영상 기반 콘텐츠를 우선한다.

제외:

- 텍스트 순간 기록

이유:

- 순간 기록은 48시간짜리 가벼운 말이므로 인기 콘텐츠로 띄우면 피드 품질이 흔들릴 수 있다.
- 인기 글 영역은 스토맵의 좋은 기록을 보여주는 쇼케이스 역할이 더 강하다.

정렬 기준 후보:

- 좋아요 수
- 댓글 수
- 최근 반응
- score

디자인 감정:

> "사람들이 많이 본 이야기는 뭐지?"

### 최신 글

최근 올라온 스토리를 보여준다.

노출 콘텐츠:

- 사진 스토리
- 숏츠 스토리
- 텍스트 순간 기록

최신 글은 현재성을 보여주는 영역이므로 순간 기록까지 포함한다.

정렬 기준:

- createdAt 최신순

디자인 감정:

> "방금 올라온 이야기가 있네."

### 순간 기록

사진 없이 남긴 48시간 기록만 모아볼 수 있는 모드다.

노출 콘텐츠:

- 텍스트 순간 기록만

1차에서는 별도 칩으로 넣을지, 카드 영역 안에서 섞어 보여줄지 보류한다.

### 버튼별 콘텐츠 노출 기준 요약

| 버튼 | 사진 스토리 | 숏츠 스토리 | 텍스트 순간 기록 | 기준 |
|---|---|---|---|---|
| 내 근처 글 | O | O | O | 주변성 우선 |
| 인기 글 | O | O | X | 볼거리/기록 품질 우선 |
| 최신 글 | O | O | O | 현재성 우선 |
| 순간 기록 | X | X | O | 48시간 텍스트 전용 |

## 6. 기존 카드 흐름 유지

이번 작업에서 유지할 것:

- 마커 선택 시 기존 하단 카드 노출
- 선택 카드 좌우 스와이프
- 좌우 스와이프하면서 옆 마커/스토리 보는 흐름
- 확장 카드
- 댓글 영역

이번 작업에서 바꾸지 않을 것:

- 선택 카드 전체 구조
- 확장 카드 전체 구조
- 댓글 UI
- 마커 선택 후 카드 동작

---

## 7. 피해야 할 것

- 지도탭에 `내 글 보기`를 넣지 않는다.
- 지도탭을 내 기록 관리 화면처럼 만들지 않는다.
- 발견 카드 슬라이드만으로 지도 카메라가 이리저리 움직이게 하지 않는다.
- 첫 진입 카드 영역을 너무 높게 만들어 지도를 가리지 않는다.
- 필터 버튼을 게시판 탭처럼 무겁게 만들지 않는다.

---

## 8. 디자인 체크리스트

디자인 시 확인할 것:

- 앱 진입 1초 안에 "내 주변 이야기"가 보이는가?
- 검색바보다 발견 카드가 더 스토맵답게 느껴지는가?
- `내 근처 글`, `인기 글`, `최신 글` 버튼이 지도 위에서 자연스러운가?
- 첫 진입 카드 영역이 지도 탐색을 방해하지 않는가?
- 텍스트 순간 기록과 사진 스토리가 서로 다른 성격으로 보이는가?
- 선택 카드/확장 카드와 새 발견 카드가 헷갈리지 않는가?

---

## 9. 보류 결정

아래는 디자인 시안 또는 실제 사용 흐름을 보고 결정한다.

- `순간 기록` 칩을 1차에 넣을지 여부
- 카드 높이 정확한 비율
- `내 근처 글` 정렬 기준
- 발견 카드 탭 시 지도 카메라를 이동할지 여부
- 마커 선택 강조 방식
- 카드 영역 이름 최종안
