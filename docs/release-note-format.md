# 출시노트 작성 포맷 (표준)

스토맵 출시노트는 **항상 아래 포맷**으로 작성한다. 일관성 유지가 목적.

## 포맷

```
[<버전> 업데이트]

• <업데이트 항목>
• <업데이트 항목>
• ...
```

- 첫 줄: `[<버전> 업데이트]` (예: `[1.3.0 업데이트]`)
- 빈 줄 1개
- 그 아래 업데이트 항목을 `•` 불릿으로 나열

## 작성 규칙

- 사용자 관점, 한국어. 기능 이름보다 **무엇이 좋아졌는지** 위주로.
- 한 줄 = 한 항목. 짧고 명료하게.
- 내부 작업(.gitignore, chore, 리팩터 등 사용자 체감 없는 것)은 제외.
- 범위: 직전 출시 버전 이후의 커밋. (마지막 출시 표기 커밋 기준)
- 버전은 `pubspec.yaml`의 `version` 기준.
- 안정성/버그 수정은 묶어서 마지막 항목으로.

## 예시 (1.3.0)

```
[1.3.0 업데이트]

• 마이페이지 전면 개편 — 내 스토리·좋아요한 스토리를 한눈에
• 로그인·회원가입 화면 개편 및 다크모드 지원
• 스토리 공유 화면을 지도 위 바텀시트로 개편
• 지도 카드·마커 사용성 개선, 텍스트 스토리 표시 개선
• 내 글 수정·삭제 플로우 통합
• 로그인·스플래시·마커 안정화, 앱 아이콘 교체
```
