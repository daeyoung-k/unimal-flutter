# 나만의 지도 (My Map) 설계 — 무엇을 / 왜

> 작성: 2026-06-29.
> 상위 방침: `docs/architecture/마이페이지-나만의지도-정보구조.md` (IA 결정).
> 구현 계획: `docs/plans/2026-06-29-나만의지도.md` (한 쌍).

## 한 줄 요약

내 스토리를 지도 위에서 보는 **별도 콘텐츠 화면 "나만의 지도"**를 신설한다. 마이페이지(허브)는 그대로 두고, 마이페이지와 지도 탭 두 곳에서 진입한다. 같은 화면을 `isOwner`로 owner/visitor 모드 분기한다.

## 왜 (배경)

마이페이지를 지도화하려다, 마이페이지는 설정·공지·좋아요·팔로우 같은 **기능 허브(목록 성격)**라 지도에 못 담는다는 결론. 지도는 "내 스토리의 공간적 표현"이라는 콘텐츠 뷰다. 둘은 그릇이 다르므로 분리한다. (자세한 근거는 architecture 문서)

## 화면 1 — 나만의 지도 (owner / visitor)

같은 화면 하나, `isOwner` 분기. 피그마: 작업영역 파일 "마이페이지 지도화" 섹션 (owner `149:8`, visitor `163:135`).

### 공통 레이아웃 (지도 위 레이어)
- **베이스**: 네이버 지도 (화이트/옅은 톤). 디자인 토큰 `background`.
- **마커**: 내 스토리 = 사진 썸네일 마커. **동선(polyline) 없음.** 선택 시 파란 링 강조.
  - 컴포넌트: 라이브러리 `[지도] 커스텀 마커` + 선택 강조는 `marker-as-strip`(`2026-05-16-marker-as-strip.md`)의 링 합성 패턴 재사용.
- **프로필 헤더 (좌상단)**: 흰 알약형 칩 — 아바타 + 닉네임 + 스토리 수. 지도 위에 떠 있음("프로필=지도 저자" 구도).
- **카메라**: 화면 진입 시 **내 스토리들의 bounds에 fitBounds.** "내 영역" 소유감의 핵심. (정적 시안엔 없고 구현 동작)
- **하단 시트**: 끌어올리는 바텀시트. 마커 탭 시 게시글 상세는 기존 `PostDetailSheet` 재사용.
- **탭바**: 라이브러리 `네비게이션 바 (마이페이지)`.

### owner 뷰 (내가 볼 때)
- 우상단: **설정 기어**.
- 비공개 스토리 포함.
- 하단 시트: **세그먼트 토글 `내 스토리 N / 좋아요한 N`** + 리스트 행(썸네일·제목·위치·시간·좋아요).

### visitor 뷰 (친구가 볼 때 — 추후 팔로우 기능과 함께)
- 우상단: 설정 기어 → **공유 + 팔로우 버튼**.
- 공개 스토리만.
- 하단 시트: **`스토리 N개` 헤더** + 리스트 행 (세그먼트 없음).

## 화면 2 — 진입점 (입구 둘, 화면은 공유)

### 진입 A · 지도 탭 → 내 지도
피그마: Entry A `168:189`.
- 지도 탭 상단엔 이미 **검색바 + 카테고리 칩 줄**이 있음.
- **검색바 우측 프로필 아바타**(지도 핀 뱃지)를 탭 → 나만의 지도(owner). 지도·네이버 앱의 "내 공간" 진입 패턴.
- 카테고리 칩 줄은 순수 필터로 유지 (내 지도 진입을 칩에 섞지 않음).
- ⚠ 칩 라벨(전체/맛집/카페/풍경/야경)·검색 placeholder는 **플레이스홀더** — 스토맵 실제 값으로 교체 필요.

### 진입 B · 마이페이지 → 내 지도 (확정)
피그마: Entry B `165:162`.
- 마이페이지 허브 구조(히어로 카드 / 메뉴 / 더보기 / 네비바)는 유지.
- 기존 "내 스토리" 가로 썸네일 스트립을 **지도 미리보기 큰 버튼(map-card)**으로 대체.
  - 미니 지도 + 사진 마커 + 하단 그라데이션 + `내 지도 보기` + `N개의 스토리 · 지역` + 화살표.
  - 탭 → 나만의 지도(owner).

## 디자인 토큰 / 컴포넌트 (피그마 라이브러리 「[스토맵] 디자인 정보」)

| 자산 | 키 / 메모 |
| --- | --- |
| 네비게이션 바 (마이페이지) | `c6a940ec...` — 402×100, 마이페이지 활성 |
| [지도] 커스텀 마커 | `89017b38...` — 36×52, `Container`=사진 슬롯, 라벨 텍스트 |
| AppColors 변수 | `background` `backgroundPage` `textPrimary` `textMuted` `primaryStrong` (+ Light/Dark) |
| 아이콘 | settings, favorite, place, pin_drop, map 등 |

### ⚠ 라이브러리 갭 (별도 보강 필요)
- **AppColors 변수가 5개뿐.** `surface`·`border`·`textSecondary`·`accentCoral` 등은 변수 없음 → 시안은 코드 토큰과 같은 hex로 채움(Flutter `app_colors.dart`와 1:1).
- **'지도' 활성 네비바 변형 없음** (마이페이지용만). Entry A 탭바는 손그림.
- 커스텀 마커가 흰 링 없는 납작한 원형 → 지도에서 덜 튐.

## 미해결 / 확인 필요
1. **백엔드**: "내 스토리만" 마커 조회 API. 기존 `GET /board/map/location/post`(score 정렬, 전체)에 작성자 필터 또는 신규 엔드포인트 필요. visitor용은 특정 memberId의 **공개** 스토리만.
2. **공개/비공개** 스토리 구분 필드 존재 여부 (visitor 필터에 필요).
3. 팔로우 기능은 추후 — visitor 경로는 그때 활성화.
4. 지도 탭 카테고리 칩·검색 문구 실제 값.
