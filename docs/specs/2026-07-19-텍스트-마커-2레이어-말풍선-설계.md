# 텍스트 마커 2-레이어 말풍선 설계

- 작성일: 2026-07-19
- 상태: Phase 1 검증 완료·커밋(13892ec), Phase 2 구현 완료(검증 대기), Phase 3 미착수
- 대상: `lib/screens/map/map_naver.dart`
- 우선 검증 플랫폼: iOS
- 선행 문서: `docs/specs/2026-07-17-텍스트-마커-로컬-전환-복구-design.md`
  (in-place·payload 재생성 시도의 실패 기록)

## 1. 목표

1. 점(일반 텍스트) ↔ 말풍선 마커 전환이 **부드럽게** 된다 (페이드).
2. 줌 인(17.3 이상)하면 말풍선 마커가 보인다.
3. 줌 아웃(16.8 미만)하면 점 마커로 돌아간다.

## 2. 확정된 제약 (2026-07-14 ~ 07-18 검증 결과)

이 설계는 아래 세 가지를 전제로 한다. 위반하는 구현은 채택하지 않는다.

- **C1. 클러스터러블 되돌림**: `NClusterableMarker`는 리클러스터링(카메라
  이동·줌·오버레이 추가)마다 최초 add payload 의 아이콘·크기·캡션으로
  leaf 를 다시 그린다 (iOS, flutter_naver_map 1.4.4, #276 유사).
  → 클러스터러블 마커에 대한 in-place `setIcon`/`setAlpha` 전환은 불가능.
- **C2. delete/add 핸들러 경합**: 플러그인 `deleteOverlay` 는 네이티브 ack
  이후 Dart 탭 핸들러를 제거하고 `addOverlayAll` 은 호출 즉시 등록한다.
  같은 id 를 await 없이 delete→add 하면 새 핸들러가 지워져 탭이 죽는다.
- **C3. 재조회 중 오버레이 추가 프리즈**: 재조회 진행 중 클러스터러블
  오버레이 추가가 네이티브 클러스터러의 dispatch_sync 와 겹쳐 앱 프리즈를
  재현한 이력 (2026-07-16). → 레이어 조작은 `_isLoadingMarkers` 중 금지.

## 3. 아키텍처: 2-레이어 분리

**점은 클러스터 세계, 말풍선은 일반 오버레이 세계.**

### 3.1 점 레이어 (기존 `NClusterableMarker`)

- payload 는 **영원히 점**. 텍스트 마커에 대한 표현 분기
  (`textCardMode`, `_textMarkerCardMode`, suppressBubble 빌드 분기)를
  updateMarkers 에서 **전부 삭제**한다.
- in-place 변경이 없으므로 C1 되돌림이 원천적으로 무의미해진다.
- 재생성은 스택 글 수 변화 등 기존 사유만 남는다.

### 3.2 말풍선 레이어 (신규, 일반 `NMarker`)

- id: `bubble_{postId}` — 점과 다른 id 네임스페이스. 같은 id 재생성이
  없으므로 C2 경합 없음.
- 일반 NMarker 는 클러스터러가 관리하지 않으므로 **alpha 트윈이 안전**
  (C1 비적용). 부드러운 전환은 이 레이어에서만 수행한다.
- 아이콘: 기존 `_buildTextCardIcon` (카드 + 아래 점 앵커 합성) 재사용.
  말풍선의 점 그래픽이 밑의 점 마커를 정확히 덮는다.
- zIndex: `300000 + score` — 점 레이어(200000+score)보다 항상 위.
- 탭 리스너: 기존 단일 마커와 동일한 `_selectMarker` 흐름.

### 3.3 표시 규칙

- 전역 모드: 줌 히스테리시스 유지 — 진입 `17.3`, 이탈 `16.8`
  (`TextMarkerPresentationPolicy` 재사용 가능).
- 말풍선 대상: 사진 없는 텍스트 글 + 단일 그룹 + 비밀집(dp 기준) +
  바텀 카드 억제 아님 (기존 규칙 그대로).
- 말풍선 목표 집합 = 전역 모드가 카드일 때의 eligible 집합. 아니면 공집합.

### 3.4 동기화 (단일 경로)

`_syncBubbleLayer(rawZoom, latitude)` 하나로 수렴:

1. 히스테리시스로 전역 모드 갱신
2. `_isLoadingMarkers` 면 반환 (C3) — 재조회 종료 후 수렴 패스가 재호출
3. 목표 집합 계산 → 현재 표시 집합과 diff
4. 추가분: 아이콘 비동기 생성 → 세대 검증 → `addOverlayAll`(alpha 0)
   → 페이드 인
5. 제거분: 페이드 아웃 → `deleteOverlay`

트리거: ① `_onCameraIdle` ② 재조회 성공 직후 1회 ③ 카드 닫힘 복원.
셋 다 같은 함수. 불일치 없으면 no-op.

### 3.5 경합 방어

- 표현 세대(latest-wins): 카메라 이동·새 sync 시작마다 증가, 비동기
  아이콘 완료 시 검증 후 폐기.
- 페이드 타이머: id 별 1개, 재전환 시 이전 타이머 취소 후 현재 alpha 에서
  이어서 진행 (사라지는 중 → 다시 나타나기 자연스럽게).
- 재조회로 글이 사라지면(뷰포트 이탈·삭제) 해당 말풍선도 제거.
- 다크모드 forceRebuild 시 말풍선 레이어도 전체 재구성.

### 3.6 카드 열림 억제 (Phase 3)

- 카드 열림: 말풍선 레이어 페이드 아웃 → 삭제 (점은 원래 항상 있으므로
  별도 복원 상태·restoreIds 불필요 — 기존 `_syncTextBubbleSuppression`
  의 in-place 억제/복원 로직 대체 가능성 검토).
- 카드 닫힘: `_syncBubbleLayer` 재호출로 현재 줌 기준 재구성.

## 4. 오류 처리

- 아이콘 생성 실패: 해당 글 말풍선 생략(점은 그대로 보임) — 마커 소실 없음
- addOverlayAll 실패: 표시 집합 미갱신 → 다음 sync 가 재시도
- 세대 불일치: 오류 아님, stale 폐기

## 6. 단계별 구현 계획

각 Phase 끝마다 사용자 실기기 검증 후 다음 단계 진행. 커밋은 승인 후.

- **Phase 1 — 전환 신뢰성**: 말풍선 레이어 추가/제거 (애니메이션 없음).
  updateMarkers 의 텍스트 카드 분기 제거. 검증: 17.3/16.8 전환, 같은 API
  줌 구간 전환, 빠른 줌 반복, 탭 동작, 재조회 후 유지.
- **Phase 2 — 부드러움**: 페이드 인/아웃(kBubbleFadeDuration 200ms).
  충돌 숨김(Q1)은 페이드와 교차 — 인 완료 후 켜고, 아웃 시작 전에 꺼서
  점이 말풍선 밑에서 자연스럽게 나타나고 사라진다. 페이드 아웃 중 재목표
  시 취소 후 현재 alpha 에서 복귀. 탭 우선순위 확인(Q2)은 실기기.
- **Phase 3 — 상호작용**: 카드 열림 억제/닫힘 복원, 기존 suppression
  로직 정리, 계약 테스트 재작성.
