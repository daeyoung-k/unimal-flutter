# 우리지도(폐쇄형 공유 지도) — Flutter UX 설계

> 네이밍 확정: **우리지도** (2026-07-11, 구 "비밀지도"). 코드 식별자(`secret_map`)는 유지, UI 표기만 변경.

> 작성일: 2026-07-11 · 상태: 초안 · 관련: unimal-server `docs/specs/2026-07-11-비밀지도-설계.md` (데이터 모델·API)

## 1. 컨셉

- 메인 지도 = 모두가 스토리를 쌓는 공개 공간 (기존 그대로)
- 우리지도 = 초대된 멤버끼리만 보는 지도. "친구와 함께 지도를 가꾸는 느낌"
- 내지도 = 나만 보는 내 글 아카이브 (우리지도 글 포함, 출처 표시)

## 2. 네비게이션 구조

```
루트 탭
├── 지도 (메인, 기존)
├── 우리지도 (신규 탭)
│   ├── [뎁스 1] 내 지도 리스트
│   └── [뎁스 2] 지도 화면 (마커 뷰 ⟷ 게시판 뷰)
├── 작성 (기존, 지도 선택 추가)
└── 마이페이지 (기존)
```

## 3. 화면별 설계

### 3.1 우리지도 리스트 (뎁스 1)

- 카드형 리스트. 각 카드: **미니 지도 프리뷰**(마이페이지 간이 지도 패턴 재사용, 사이즈 축소) + 지도명 + 멤버 수 + 최근 활동 시각
  - 예: `광진구 런닝지도 · 멤버 12 · 3시간 전`
  - 미니 지도는 서버가 내려주는 마커 좌표 목록으로 정적 렌더 (인터랙션 없음)
- 상단 액션: `+ 지도 만들기`, `초대 코드로 참여`
- 빈 상태(지도 0개)가 첫인상 — 온보딩 카피 + 예시 지도명 노출("길고양이 급식소 지도, 러닝 크루 코스…") + 만들기/참여 CTA 두 개
- 새 글 있는 지도는 뱃지 표시

### 3.2 지도 화면 (뎁스 2)

- 기본은 **마커 뷰**: 기존 지도 화면(네이버 지도 + bottom_card) 패턴 재사용. 마커/클러스터링은 공용 상수화된 마커 시스템 그대로
- 상단 앱바: 지도명 + 멤버 아바타 스택 + 더보기(멤버 관리·초대·설정)
- **게시판 뷰 토글**: 마커 없는 글(담소 TALK, 공지 NOTICE) 리스트. NOTICE 상단 고정
- FAB: 글쓰기 → `스토리(위치 필수)` / `담소(위치 없음)` 선택. 공지는 방장에게만 노출
- 방장 전용: 초대 코드 공유(카톡 딥링크), 멤버 강퇴, 지도 수정/폐쇄

### 3.3 글 작성 (기존 add/ 확장)

- 작성 진입 컨텍스트에 따라 대상 지도 자동 선택 (우리지도 화면에서 진입 시 해당 지도 고정)
- 메인 탭에서 진입 시 기본값 = 메인 지도, 드롭다운으로 내 우리지도 선택 가능
- `postType`: 우리지도 대상일 때만 스토리/담소 선택 UI 노출

### 3.4 내지도 (기존 확장)

- 내가 쓴 모든 글 표시 (메인 + 우리지도)
- 마커/카드에 출처 뱃지: 메인은 무표시, 우리지도 글은 지도명 칩
- 상단 필터 칩: `전체 | 메인 | 지도별` — 상태는 로컬 유지
- 지도 폐쇄된 글도 내지도에는 계속 표시 (서버 정책과 동기화)

### 3.5 초대 플로우

- 공유: 방장이 `초대 링크 공유` → 카톡 등으로 `https://unimal.co.kr/m/{code}` 전송
- 수신(설치됨): 링크 탭 → 앱 열림 → 합류 확인 시트(지도명·멤버 수·방장) → 참여
- 수신(미설치): 웹 랜딩 → 스토어 → 설치 후 리스트 화면의 `초대 코드로 참여`에 코드 수동 입력 (deferred deep link는 MVP 제외)
- 앱링크/유니버설링크 설정 작업 필요 (Android intent-filter, iOS AASA)

### 3.6 푸시 랜딩

- `[지도명] ○○님이 새 스토리를 남겼어요` 탭 → 해당 우리지도 화면으로 이동 (스토리면 해당 마커 포커스, 담소/공지면 게시판 뷰)

### 3.7 탐색 · 참여 신청 (신규 제안, 피그마 시안 완료)

- 우리지도 탭에 `내 지도 | 탐색` 세그먼트 추가
- **핫한 우리지도 리스트**: 랭킹형 카드 — 순위, 1위 HOT 뱃지(`accent`), 지도명, 소개 한 줄, `멤버 n · 이번 주 새 글 n`, 참여 신청 버튼 (신청 후 "대기중" 비활성)
- **참여 신청 시트**: 미니맵 배너 + 지도명 + 메타 + **방장 소개글**(primaryWash 박스, 방장 아바타·뱃지) + 멤버 아바타 스택 + CTA. 캡션: "방장이 수락하면 지도에 참여할 수 있어요"
- **신청 완료 화면**: 승인 대기 안내 + 요약 카드("승인 대기중" 칩) + 신청 취소
- 서버 스펙 영향 (unimal-server 설계 문서에 반영 필요):
  - 지도 공개여부 플래그 (비공개=초대 코드 전용 / 공개=탐색 노출 + 신청·승인)
  - 지도 소개글 필드 (방장 작성)
  - 참여 신청/승인/거절 상태 및 API, 신청 수락 시 푸시
  - 핫 랭킹 산출 기준 (주간 새 글 수·신규 멤버 수 등)
- 참여 경로가 2트랙(초대 코드 즉시 참여 / 공개 탐색 신청·승인)이 됨을 전제로 UX 설계

## 4. 디자인 토큰 준수

- 신규 화면 전부 `AppColors.of(context)` 시멘틱 토큰만 사용, 다크모드 기본 대응
- 화이트 기반 + 블루는 액센트만 — 지도 카드 프리뷰 테두리는 `border`, 새 글 뱃지는 `primary`
- 새 시멘틱 토큰 필요 시 `app_colors.dart`에 추가 후 사용 (임의 색 금지)

## 5. 구현 순서 (제안, 서버 API 완성 후)

1. 우리지도 탭 + 리스트 화면 (빈 상태 포함)
2. 지도 생성/참여(코드 입력) 플로우
3. 지도 화면 마커 뷰 (기존 지도 컴포넌트 재사용)
4. 글 작성 확장 (대상 지도 + postType)
5. 게시판 뷰 (담소/공지)
6. 내지도 출처 뱃지 + 필터
7. 딥링크 + 푸시 랜딩
