# 스펙: 지도 카드 + 썸네일 스트립 리디자인

> 작성일: 2026-05-13  
> 상태: 승인됨

---

## 1. 목표 및 핵심 원칙

**"지도에서 모든 걸 다 하자"** — 상세 페이지로 이동하지 않고 지도 위에서 게시글 조회, 이미지 탐색, 댓글까지 모두 처리한다.

기존 peek/full 바텀시트 + 상하 스와이프 탐색 구조를 완전히 교체한다.

---

## 2. 두 가지 포스트 타입

### 2-1. 이미지 포스트 (영구 아카이브)

- 이미지를 포함한 게시글
- 지도에 **영구 저장**, 삭제 전까지 마커 유지
- **카드 높이: 화면의 ~62%**
- 마커 핀이 카드 아래에 가려짐 (의도된 동작)
- 썸네일 스트립에서 사진 썸네일로 표시

### 2-2. 텍스트 포스트 (휘발성)

- 제목·내용만 올린 게시글 (이미지 없음)
- **24시간 후 자동 소멸**
- **카드 높이: 화면의 ~38%**
- 카드가 작아서 마커 핀이 지도에 보임 (자연스러운 노출)
- 썸네일 스트립에서 💬 아이콘 + 주황 테두리로 표시

---

## 3. 전체 화면 레이아웃

```
┌─────────────────────────────┐
│  🔍 장소, 이야기 검색        │  ← 검색바 (항상 고정)
│  [전체] [친구만] [내 글]     │  ← 필터 칩 (항상 고정)
│                             │
│         지 도               │  ← 지도 (가변 높이)
│    (마커: 이미지 포스트엔     │
│     카드 아래 숨겨짐)        │
│                             │
│  +2 [□][□] [■] [□][□] +3   │  ← 썸네일 스트립
├─────────────────────────────┤
│                             │
│         카 드               │  ← 카드 (62% 또는 38%)
│                             │
└─────────────────────────────┘
```

- 검색바 + 필터는 항상 최상단 고정
- 스트립은 카드 바로 위에 위치하며 카드가 확장될 때 필터 영역 하단에 고정됨 (더 위로 올라가지 않음)
- 마커 핀은 항상 지도 중앙에 위치, 이미지 포스트 카드는 마커를 덮어 가림

---

## 4. 썸네일 스트립 (고정 윈도우 방식)

### 구조
```
+N  [□] [□]  [■]  [□] [□]  +N
     ↑    ↑   ↑    ↑    ↑
     좌2  좌1  현재  우1  우2
```

- 항상 **현재(크게) + 좌우 각 2개** 고정 표시
- 범위 밖 마커는 `+N` 숫자로만 표시
- 현재 마커: 더 크게(~36×36px), 파란 테두리 + glow
- 인접 마커: 작게(~26×26px), 거리에 따라 opacity 감소 (좌1/우1: 65%, 좌2/우2: 50%)
- 텍스트 포스트: 주황 테두리, 💬 아이콘
- **탭**: 해당 마커로 바로 점프 + 카메라 fly 이동

### 스트립 위치 규칙
| 카드 상태 | 스트립 위치 |
|---------|-----------|
| 기본 | 카드 상단 바로 위에 浮遊 |
| 확장 중 | 카드와 함께 위로 이동 |
| 필터 영역 도달 시 | 필터 하단에 고정 (더 이상 위로 못 올라감) |
| 카드 닫힘 | 스트립도 카드와 함께 사라짐 (화면 밖으로) |

---

## 5. 카드 3단계 상태

| 상태 | 높이 | 진입 방법 | 탈출 방법 |
|------|------|---------|---------|
| **닫힘** | 0% | 아래로 드래그 | 마커 탭 |
| **기본** | 62% (이미지) / 38% (텍스트) | 마커 탭 | 위로 드래그 / 아래로 드래그 |
| **확장** | ~88% | 핸들 위로 드래그 | 아래로 드래그 |

- 스냅: 기본 ↔ 확장 두 위치만. 중간에 손을 놓으면 가까운 쪽으로 스냅
- 닫힘 → 기본: 마커 탭 시 fly 애니메이션(600ms)으로 카메라 이동 + 카드 슬라이드 업

## 6. 카드 내부 구조

### 6-1. 이미지 포스트 카드 (기본 상태)

```
┌─ 핸들 ──────────────────────┐
│ ┌─────────────────────────┐ │
│ │   이미지 캐러셀 영역      │ │  ← 고정 높이
│ │  ‹              ›        │ │  ← 탭 존 (이미지 내 좌우)
│ │          ●○○○            │ │  ← 이미지 dot indicator
│ └─────────────────────────┘ │
│  제목                        │
│  📍 위치 · 시간 · ♥ N 💬 N  │
│  내용 미리보기 (2줄)...더보기 │
│                  ↑ 드래그   │
└─────────────────────────────┘
```

### 6-2. 이미지 포스트 카드 (확장 상태)

```
┌─ 핸들 ──────────────────────┐
│ ┌─────────────────────────┐ │
│ │   이미지 캐러셀 (고정)    │ │  ← 스크롤해도 고정
│ └─────────────────────────┘ │
│ ─ ─ ─ ─ 스크롤 영역 ─ ─ ─ ─│
│  제목                        │
│  📍 위치 · 시간 · ♥ N        │
│  전체 내용 텍스트             │
│  ─────────────────────────  │
│  댓글 N개                    │
│  [프로필] 닉네임 · 시간       │
│  댓글 내용                   │
│  [프로필] 닉네임 · 시간       │
│  댓글 내용                   │
│  ...                         │
│ ─ ─ ─ ─ 스크롤 끝 ─ ─ ─ ─  │
├─────────────────────────────┤
│ [아바타]  나도 한 마디...  ›  │  ← 댓글 입력 (고정)
└─────────────────────────────┘
```

### 6-3. 텍스트 포스트 카드

```
┌─ 핸들 ──────────────────────┐
│ [아바타] 닉네임  시간  [24h] │
│  텍스트 내용                 │
│  ♥ N  💬 댓글               │
└─────────────────────────────┘
```

- 확장 시 댓글 스크롤 동일하게 적용

## 7. 제스처 모델

| 제스처 | 위치 | 동작 |
|--------|------|------|
| 카드 핸들 위로 드래그 | 핸들 | 기본 → 확장 |
| 카드 핸들 아래로 드래그 | 핸들 | 기본/확장 → 닫힘 |
| 카드 좌우 스와이프 | 카드 전체 (이미지 영역 제외) | 이전/다음 마커로 이동 + 카메라 fly |
| 이미지 영역 ‹ 탭 | 이미지 좌측 28px | 같은 게시글 이전 사진 |
| 이미지 영역 › 탭 | 이미지 우측 28px | 같은 게시글 다음 사진 |
| 스트립 썸네일 탭 | 스트립 | 해당 마커로 점프 + 카메라 fly |
| 확장 카드 내부 드래그 | 스크롤 영역 | 내용/댓글 스크롤 |
| 경계 도달 시 | - | 햅틱 피드백 (lightImpact) |

### 제스처 충돌 해결
- 이미지 캐러셀: `ClampingScrollPhysics` (PageView) → 좌우 스와이프 흡수, 카드 좌우 스와이프와 분리
- 확장 카드 내부 스크롤: 수직 드래그 흡수 → 카드 핸들 아래로 드래그와 구분 (핸들 전용 drag 영역)
- 카드 좌우 스와이프: GestureDetector on 카드 레벨 (이미지 PageView 영역 제외)

---

## 8. 카메라 이동

- **마커 탭** → `NCameraUpdate.scrollAndZoomTo(target, zoom: 16)`, fly 600ms
- **스트립 탭** → 동일
- **카드 좌우 스와이프** → 다른 마커 그룹으로 전환 시 fly 600ms, 같은 그룹 내 게시글 전환 시 이동 없음
- 줌은 마커 탭 시에만 16으로 고정, 나머지는 현재 줌 유지

## 9. 기존 코드와의 관계

### 재사용
- `PostGroupNavigator` — 그룹 순회 로직 (next/prev/jumpToGroup) 그대로 활용
- `PostImageCarousel` — 이미지 캐러셀 위젯 재사용 (높이 조정)
- `PostInfoSection` — 내용 표시 부분 재사용 (레이아웃 변경)
- `relative_time.dart` — 그대로 사용

### 교체
- `MapBottomCard` — 전체 재작성 (peek/full → 기본/확장 3단계 + 스트립)
- `map_naver.dart` 카드 통합 부분 — 새 카드 위젯으로 교체

### 신규
- `MapThumbnailStrip` — 고정 윈도우 스트립 위젯
- `MapCardScrollable` — 확장 상태 내부 스크롤 영역
- 이미지/텍스트 포스트 타입 분기 로직

## 10. 미결정 사항 (구현 시 결정)

- 스트립 썸네일 크기 (현재: 36px 활성 / 26px 비활성) — 기기별 테스트 후 확정
- 카드 확장 snap 임계값 (드래그 거리 및 속도) — 기존 60px/300px/s 기준으로 시작
- 텍스트 포스트 확장 최대 높이 (댓글 많을 때) — 88% 동일 적용 예정
- 댓글 입력 시 키보드 올라올 때 카드 높이 처리
