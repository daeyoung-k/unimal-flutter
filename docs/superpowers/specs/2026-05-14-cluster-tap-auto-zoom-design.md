# 클러스터 마커 탭 시 줌 15+ 자동 확대

> 작성일: 2026-05-14
> 관련 파일: `lib/screens/map/map_naver.dart`
> 관련 메모: `stomap-zoom-and-clustering-policy.md`

## 배경

지도에서 줌 12 같은 낮은 줌 레벨에선 같은 영역의 마커가 클러스터로 묶임. 클러스터 마커를 탭하면 라이브러리 기본 동작이 1순위 마커로 가볍게 줌인하는 정도라, 클러스터가 완전히 풀리지 않은 채 사용자가 1순위 마커를 다시 탭해 카드/스트립을 열게 됨.

이때 스트립(`_postGroups` 전체 표시)에는 시각적으로 여전히 클러스터에 묶여 안 보이는 마커의 콘텐츠도 포함됨. 사용자가 그 하위 콘텐츠를 스트립으로 선택하면 카드 본문은 바뀌지만 지도 상의 마커는 1순위 클러스터 아이콘 그대로라, "내가 보는 콘텐츠가 어디 있는지 모름" 상태가 됨.

스트립에 멤버를 포함하는 동작은 의도된 동작(정보 우선)이므로 유지. 대신 클러스터 진입 시점에 줌인을 충분히 해서 클러스터링을 자동으로 해제하면 정합성이 자연스럽게 회복됨.

## 설계

**핵심 변경:** `_buildClusterMarker(info, clusterMarker)` 안에서 `clusterMarker.setOnTapListener(...)`로 클러스터 탭 핸들러를 부착한다.

**탭 시 동작 (`info.size > 1` 일 때만):**

1. `info.children` 에서 score 최대 child 의 position 을 1순위 좌표로 추출.
2. `_mapController.updateCamera(NCameraUpdate.scrollAndZoomTo(target: 1순위좌표, zoom: max(현재줌, 15)))`.
   - 현재 줌이 15+ 면 좌표 이동만, 14 이하면 15까지 확대.
3. 줌 15+ 진입 → 기존 클러스터 정책에 따라 클러스터링 자동 비활성 → jitter 멤버들이 펼쳐짐.
4. `_onCameraIdle` 가 자동 재조회 (기존 디바운스 로직 그대로) → 새 영역의 마커 데이터 갱신.

**자동 카드 노출은 하지 않음.** 클러스터 탭 = 줌인까지만. 카드는 사용자가 펼쳐진 마커를 한 번 더 탭해야 등장 (기존 2-step 동선 유지).

**좌표 선택 — 1순위 마커 좌표 사용.** 현재 클러스터 아이콘 이미지가 1순위 마커 그림이라 "탭한 자리 = 카메라가 가는 자리"가 시각적으로 일치. `_buildClusterMarker`가 이미 topId/topScore를 계산하므로 그 자리에서 함께 추출.

## Edge cases

- `info.size == 1` (단일 마커가 클러스터 빌더를 거치는 경우): onTap 미부착. `NClusterableMarker` 자체의 onTap (기존 마커 핸들러)이 동작함.
- 이미 줌 15+ 인 상태: 정책상 클러스터링 OFF이므로 발생 가능성 낮음. 방어 코드로 줌 인하지 않고 좌표 이동만.
- `setOnTapListener`가 `_buildClusterMarker` 매 호출마다 새로 부착됨 → 동일 콜백 형태(closure)로 매번 갱신되므로 누수/중복 호출 없음.

## 비변경 영역

- 줌 정책 (기본 15, 클러스터 0–14) — 직전 변경 그대로.
- `_postGroups` 구조 / 스트립 표시 정책 ("정보 우선") — 유지.
- 스트립 좌우 스와이프, 카드 인터랙션 — 유지.
- `_moveCameraToMarker` 의 25% 지점 알고리즘 — 마커 탭 동선에서만 사용. 클러스터 탭은 별도 동작이라 영향 없음.

## 후속 작업 후보 (이번 범위 밖)

- 클러스터 탭 시 1순위 카드를 자동 노출해서 2-step → 1-step 단순화.
- 줌 15까지 가도 같은 좌표 그룹이 jitter 5개 한계로 잘려 일부 멤버는 여전히 안 보일 수 있음 → 그 그룹에 대한 별도 펼침 UI.
