# iOS 마커 이미지 캐시 초기화 경합 크래시 (NOverlayImage.swift:16)

## 증상

- 핫 리스타트 직후 지도 마커 로드 중 앱 크래시 (SIGTRAP, 메인 스레드)
- 크래시 스택: `ClusteringController.onClusterableMarkerUpdate` → `NMarker.applyAtRawOverlay`
  → `NOverlayImage.makeOverlayImageWithPath()` **강제 언래핑 nil** (NOverlayImage.swift:16)
- 동반 로그: `PathNotFoundException: Deletion failed, path = '.../Caches/fnm1_img/fnm1_img_XXXXXX'`
  (이미 없는 폴더를 또 지우려다 실패 — 아래 이중 실행의 직접 증거)

## 원인 — 플러그인 `ImageUtil` 락 없는 lazy 초기화 (flutter_naver_map 1.4.4)

플러그인은 마커 이미지를 `Caches/fnm1_img/fnm1_img_XXXX/`(세션별 폴더)에 PNG로 저장하고,
네이티브가 그 **파일 경로**를 로드한다. 문제는 초기화 코드:

```dart
// ImageUtil (플러그인 1.4.4)
static Future<Directory> _getDir() async {
  if (_imageTempDir case Directory dir) return dir;   // ← 락 없음 (check-then-act)
  final imageTempDir = await _initTempDir();          // ← 동시 진입 가능
  _imageTempDir = imageTempDir;
  return imageTempDir;
}

static Future<Directory> _initTempDir() async {
  ...
  await _cleanUpPreviousTempDir(targetFolderDir);     // 기존 세션 폴더 전부 delete (unawaited!)
  final parent = await targetFolderDir.create();
  return parent.createTemp(_newPathPrefix);           // 새 세션 폴더 생성
}
```

앱 시작(핫 리스타트 포함) 직후 이미지 저장이 **동시에 2건** 들어오면 `_initTempDir()`이
두 번 실행된다:

1. A: 이전 세션 폴더 삭제 → 새 세션 폴더 **S-a** 생성 → 이후 마커 아이콘을 S-a에 저장
2. B(동시): 폴더 목록에서 방금 생긴 **S-a를 "이전 세션"으로 오인 → 삭제**(unawaited) → S-b 생성
3. S-a에 저장된 아이콘 경로를 네이티브가 로드 → 파일 없음 → nil 강제 언래핑 → **크래시**

같은 이전 폴더를 A·B가 이중 삭제하다 나온 것이 로그의 `PathNotFoundException`.
핫 리스타트는 Dart 정적 상태(`_imageTempDir`)만 리셋하고 이전 네이티브 지도의 비동기
정리가 겹치므로 재현 확률이 특히 높다. 관련 플러그인 이슈: #251 (1.4.4 기준 미해결).

## 조치 (앱 측 방어)

1. **웜업** — `warmUpOverlayImageCache()` (marker_image_factory.dart):
   main.dart에서 지도/마커 생성 전에 1x1 PNG 저장을 1회 await → temp dir 초기화가
   단일 실행으로 끝나고, 이후 모든 호출은 fast path(캐시된 Directory)를 타 경합이 사라진다.
2. **`fromWidget` 직렬화** — `overlayImageFromWidget()` 래퍼 신설, 모든
   `NOverlayImage.fromWidget` 호출(메인 지도 텍스트 카드, 내지도 카드/클러스터 버블)을
   `overlayImageFromBytes` 와 같은 체인에 태움. 위젯 아이콘이 바이트 아이콘과 동시에
   초기화/저장을 트리거하던 구멍을 막는다 (기존 백로그 항목).
3. 기존 방어 유지: 시작 시 0바이트 파일 정리(main.dart), `overlayImageFromBytes` 체인.

## 남는 리스크

- 핫 리스타트 시 "새 세션의 이전 폴더 삭제"가 **아직 살아있는 이전 네이티브 지도**의
  마커 파일을 지우는 경합은 플러그인 구조상 앱에서 못 막는다 — 개발 중(핫 리스타트)에만
  발생 가능, 실사용 콜드 스타트에서는 이전 지도가 없어 안전.
- 업스트림 리포트 권장: `_getDir()` 초기화에 락(또는 Future 캐시), 세션 폴더 삭제를
  현재 세션 생성 이후로 지연.
